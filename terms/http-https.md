# HTTP / HTTPS — 瀏覽器和網站之間的溝通規則

> **白話說：** HTTP 是寄明信片（別人看得到內容），HTTPS 是寄掛號信封（密封的，別人看不到）。

---

## 它到底是什麼？

**HTTP**（HyperText Transfer Protocol）是瀏覽器跟[伺服器](server.md)之間「傳資料的規則」。你每次打開一個網頁，瀏覽器就是用 HTTP 去跟網站要資料的。

**HTTPS** 就是 HTTP 加上「S」= Secure（安全）。差別在於 HTTPS 會把傳輸的內容[加密](encryption.md)，就算有人在中間攔截，也看不到你傳了什麼。

簡單判斷方法：看網址列開頭
- `http://` → 沒加密（現在越來越少了）
- `https://` → 有加密（瀏覽器會顯示鎖頭圖示）

## 生活比喻 / 實際例子

**寄信：**
- **HTTP** = 寄明信片。郵差、路人都可能看到你寫的內容。
- **HTTPS** = 用密封信封寄掛號。只有你和收件人能看到內容。

**你什麼時候會聽到：**
- 「我們的網站要全部改成 **HTTPS**。」
- 「瀏覽器顯示『不安全』，因為還在用 **HTTP**。」
- 「SSL 憑證到期了，**HTTPS** 會失效。」（SSL 就是讓 HTTPS 運作的加密證書）

## 為什麼要知道這個詞？

- 如果你的網站還用 HTTP，瀏覽器會顯示「不安全」的警告，使用者看到會怕
- 網站有輸入密碼、信用卡等敏感資料的頁面，**一定要用 HTTPS**
- 搜尋引擎（Google）會優先排名 HTTPS 的網站，這會影響 SEO
- 當工程師說「要裝 SSL 憑證」，就是為了讓網站用 HTTPS
