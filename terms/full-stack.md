# Full-Stack（全端） — 前端後端都會的工程師

> **白話說：** 就像餐廳裡外場內場都能做的萬能店員——既能在前面招呼客人，也能進廚房炒菜。

---

## 它到底是什麼？

軟體開發通常分成[前端和後端](frontend-backend.md)：

- **前端**：使用者看到的畫面（網頁、App 介面）
- **後端**：背後處理資料的系統（伺服器、資料庫）

**Full-Stack（全端）工程師**就是前端和後端都會的人。他可以自己一個人搞定從畫面到資料處理的完整功能。

不過「都會」不代表「都精通」。全端工程師像是瑞士刀——什麼都能做一點，但每個功能不一定比專用工具強。

## 生活比喻 / 實際例子

想像裝修房子：

- **前端工程師** = 室內設計師，負責裝潢和美觀
- **後端工程師** = 水電師傅，負責管線和結構
- **Full-Stack 工程師** = 設計和水電都會的統包師傅

統包師傅的好處是溝通快、一個人就能搞定；缺點是遇到超複雜的設計或水電，可能還是要找專家。

實際例子：
- 主管說：「這個小專案找一個 **full-stack** 就夠了，不用分前後端」
- 招募說：「我們在找 **全端工程師**，需要會 React 和 Node.js」
- PM 說：「這個功能 UI 到 API 都要改，誰能一起做？」工程師：「我是 **full-stack**，我可以整個包」

## 為什麼要知道這個詞？

- 招人時常看到「Full-Stack Engineer」，你要知道代表什麼能力
- 小團隊或新創公司特別需要全端工程師，因為一人可以當兩人用
- 跟工程師分配工作時，知道誰是全端可以更靈活安排任務
