# Monitor / Alert（監控 / 告警）— 家裡的煙霧偵測器

> **白話說：** 自動盯著系統狀態，一有異常馬上發通知，不用人 24 小時盯螢幕。

---

## 它到底是什麼？

**Monitor**（監控）是持續追蹤系統健康狀態的機制。**Alert**（告警）是當數值異常時自動發出通知。

想像家裡的煙霧偵測器——

```
Monitor（監控）：煙霧偵測器 24 小時偵測空氣中的煙霧濃度
Alert（告警）：煙霧濃度超標 → 嗶嗶嗶大聲警報！
```

**你不用一直站在廚房聞有沒有燒焦味，偵測器會幫你盯，有事再叫你。**

## 生活比喻 / 實際例子

也像你設定的手機鬧鐘或行事曆提醒——你不用一直看時間，時間到了手機會叫你。系統監控也是一樣，工程師設定好「什麼情況要通知」，系統就會自動盯著。

### 常監控的東西

- **CPU 使用率**：電腦的腦袋會不會燒掉
- **記憶體**：會不會滿了導致系統卡住
- **錯誤率**：最近是不是很多 API 回傳錯誤
- **回應時間**：使用者等太久了嗎

### 常見情境

- **DevOps 工程師：**「我設了 alert，CPU 超過 80% 會發 Slack 通知。」
- **PM：**「昨晚系統有掛嗎？」→ 「monitor 顯示凌晨 3 點錯誤率飆高。」
- **工程師：**「alert 一直響，先去看看怎麼回事。」→ 煙霧偵測器在叫了

## 為什麼要知道這個詞？

系統 24 小時在跑，不可能有人一直盯著。Monitor 和 Alert 就是自動化的守門員。當你聽到「監控有異常」或「alert 響了」，就知道系統的自動偵測器發現問題了，工程師正在處理。
