# S3（Amazon S3） — AWS 上的超大雲端儲存空間

> **白話說：** 就像一個容量無上限的 Google Drive / Dropbox，但是給程式用的——你的網站圖片、使用者上傳的檔案、資料備份，通通丟進去。

---

## 它到底是什麼？

S3 是 **Simple Storage Service** 的縮寫，是 [AWS](aws.md) 上最常用的檔案儲存服務。

它的概念很簡單：給你一個「桶子（Bucket）」，什麼檔案都可以往裡面丟——圖片、影片、PDF、[備份](backup.md)檔、[日誌（Log）](log.md)⋯⋯容量幾乎沒有上限，用多少付多少。

S3 常被用來存：
- 網站上的圖片和靜態檔案
- 使用者上傳的頭像、文件
- 資料庫的定期備份
- 系統日誌（Log）的長期存放
- 大數據分析用的原始資料（丟進 [Data Lake](data-lake-warehouse.md)）

### 什麼是 Bucket（桶子）？

Bucket 是 S3 裡的「資料夾」概念。你會建不同的 Bucket 來放不同用途的檔案：

- `my-app-images` — 放網站圖片
- `my-app-backups` — 放備份檔
- `my-app-logs` — 放系統日誌

## 生活比喻 / 實際例子

想像你開一家公司：

- **自己買硬碟** = 自己買倉庫，容量固定，滿了要再買
- **S3** = 租一個彈性倉庫，東西放多少收多少錢，永遠不會滿

實際例子：
- 工程師說：「使用者上傳的檔案存到 **S3** 上」
- 工程師說：「資料庫備份每天自動丟到 **S3**」
- PM 問：「圖片載入很慢」工程師：「**S3** 搭配 [CDN](cdn.md) 就會快很多」

## 為什麼要知道這個詞？

- 幾乎所有用 AWS 的公司都會用到 S3，是 AWS 最基礎的服務之一
- 跟儲存有關的需求（上傳檔案、存圖片、備份資料），工程師通常就是用 S3 解決
- S3 的費用也是 AWS 帳單上常見的項目
