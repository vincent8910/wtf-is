# Test Case（測試案例）— 餐廳試菜表

> **白話說：** 列出「做什麼操作、預期什麼結果」的檢查清單，一項一項確認功能正不正常。

---

## 它到底是什麼？

**Test Case** 就是一份檢查清單，上面寫著：

1. 要做什麼操作（步驟）
2. 預期會看到什麼結果
3. 實際結果是否符合預期

想像餐廳開幕前的試菜——

```
試菜表（Test Case）：
  ☐ 點一份牛排 → 預期：10 分鐘內上桌，五分熟 → ✅ 通過
  ☐ 要求加辣 → 預期：廚房收到備註 → ✅ 通過
  ☐ 同時點 10 道菜 → 預期：30 分鐘內全部上齊 → ❌ 失敗（等了 50 分鐘）
```

**Test Case 就是試菜表——一道一道測試，確認每道菜都達標。**

## 生活比喻 / 實際例子

也像新手機出廠前的品質檢查表——打電話通不通？螢幕亮不亮？充電正不正常？觸控靈不靈敏？一項一項勾，全部通過才能出貨。

### 常見情境

- **QA（品質測試人員）：**「這個功能我寫了 20 個 test case，目前通過 18 個，有 2 個 fail。」
- **PM：**「test case 寫好了嗎？」→ 檢查清單準備好了嗎？
- **工程師：**「你的 test case 可以多加幾個邊界情況嗎？」→ 檢查清單要再加幾個特殊狀況

## 為什麼要知道這個詞？

軟體上線前一定要經過測試，而 test case 就是測試的依據。PM 常常需要跟 QA 一起確認「要測哪些東西」，這時候就是在討論 test case。
