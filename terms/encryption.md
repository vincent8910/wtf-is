# Encryption（加密）— 把信鎖進保險箱

> **白話說：** 把資料打亂成別人看不懂的樣子，只有有鑰匙的人才能解開看。

---

## 它到底是什麼？

**Encryption**（加密）就是把原本看得懂的資料，用一把「鑰匙」變成看不懂的亂碼。只有拿到正確鑰匙的人，才能把亂碼還原成原本的內容。

想像你寄信給朋友——

```
沒有加密：明信片，路上任何人都能偷看內容
有加密：把信鎖進保險箱，只有你朋友有鑰匙能打開

原始內容：「密碼是 1234」
加密之後：「a8f2$kL9!xQ」 ← 外人看到的是亂碼
解密之後：「密碼是 1234」 ← 有鑰匙的人看到的
```

**Encryption 就是把信鎖進保險箱——路上被人攔截也看不到裡面寫什麼。**

## 生活比喻 / 實際例子

你每天都在用加密，只是不知道而已：

- **Line 的「端對端加密」**：你跟朋友的對話，連 Line 的員工都看不到
- **網購刷卡**：信用卡號碼加密傳輸，不怕被竊聽
- **網址前面的 `https`**：那個 `s` 就是 secure，代表這個網站有加密

### 常見情境

- **資安工程師：**「密碼一定要加密存儲，不能存明文。」→ 密碼不能直接存「1234」，要變成亂碼才存
- **PM：**「用戶的個資有加密嗎？」→ 萬一資料外洩，別人能不能直接看到？
- **工程師：**「這個 API 傳輸有加密嗎？」→ 資料傳輸過程中有沒有被保護？

## 為什麼要知道這個詞？

只要你的產品處理任何「機密資料」——密碼、信用卡號、身分證字號、個資——加密就是必須的。不加密就像把密碼寫在明信片上寄出去，任何人都能看到。這不只是技術問題，更是法律問題（個資法要求）。
