# Bug / Debug — 抓蟲大作戰

> **白話說：** Bug 就是程式裡的「蟲」——程式的行為跟預期不一樣。Debug 就是找出原因並修好的過程。

---

## 它到底是什麼？

### Bug 這個詞怎麼來的？

1947 年，工程師在操作一台巨型電腦時機器突然出問題。找了半天，最後發現——**真的有一隻蛾卡在電路板裡**。他們把那隻蛾用膠帶黏在工作日誌上，寫了一句：「First actual case of bug being found.」

從此以後，程式出問題就叫 **Bug**，修問題就叫 **Debug**。

### 用看醫生來比喻

```
🦠 Bug = 症狀
   「使用者按送出，畫面沒反應」

🔍 Debug = 看診找病因
   「什麼時候開始壞的？做了什麼操作？」

🔬 找到原因
   「原來是這行程式碼少了一個判斷」

💊 Fix（修好）
   修改程式碼 → 測試 → 部署上線
```

### Bug 不一定是程式當掉

只要「程式的行為跟預期不一樣」，就是 Bug：

| 症狀 | 算不算 Bug？ |
|------|-------------|
| 按按鈕整個網頁崩潰 | 嚴重的 Bug |
| 金額顯示 $99.999 而不是 $100 | 是，邏輯錯誤 |
| Chrome 正常但 Safari 排版亂掉 | 是，相容性問題 |
| 功能正常但要等 30 秒 | 看情況，可能是效能問題 |

## Bug 的嚴重程度分級

| 等級 | 意思 | 例子 |
|------|------|------|
| **P0** | 災難級，系統全掛 | 網站完全打不開 |
| **P1** | 緊急，核心功能壞了 | 沒辦法結帳付款 |
| **P2** | 重要，但有替代方案 | 搜尋壞了，但可以手動找 |
| **P3** | 一般，不影響使用 | 某個按鈕顏色不對 |

## 日常場景

- **「這個 bug 我在 debug 了」**
  → 白話：這個問題我正在找原因、修理中
- **「QA 回報了一個 bug」**
  → 白話：品質測試的人發現一個問題，記錄下來了
- **「這是 P1 bug，最優先處理」**
  → 白話：超嚴重的問題，要第一個修
- **「這不是 bug，這是 feature」**
  → 工程師經典玩笑話。意思是「你說的問題，其實是故意的」——但有時候就真的只是 bug

> 💡 **PM 小技巧：** 回報 Bug 時越詳細越好。「功能壞了」不夠，「在 iPhone 的 Safari 上，點送出按鈕後畫面閃一下然後沒反應」才好。工程師會感謝你的。
