# UAT（User Acceptance Testing）— 新餐廳的試營運

> **白話說：** 上線前讓真正的使用者來實際操作一輪，確認符合需求才正式上線。

---

## 它到底是什麼？

**UAT** 就是在軟體正式上線之前，請「真正會用這個系統的人」來試用，確認功能符合他們的需求。

想像一家新餐廳的試營運——

```
開幕前的試營運（UAT）：
  → 邀請一些真正的客人來吃
  → 請他們從點餐、吃飯、到結帳完整走一遍
  → 收集回饋：「菜單看不懂」「結帳太慢」「座位不舒服」
  → 把問題修好，才正式開幕
```

**UAT 就是正式開幕前的試營運——讓真人來走一遍，確認沒問題才上線。**

## 生活比喻 / 實際例子

### UAT 跟一般測試有什麼不同？

| | 一般測試（QA 做的） | UAT |
|---|---|---|
| 誰來測 | QA 工程師 | 真正的使用者（PM、客戶、業務） |
| 重點 | 有沒有 bug | 好不好用、符不符合需求 |
| 比喻 | 品管人員檢查 | 客人試吃 |

一般測試是確認「東西有沒有壞」，UAT 是確認「東西是不是你要的」。

### 常見情境

- **PM：**「功能開發完了，下週進 UAT，請業務部門來測。」
- **QA：**「QA 測試都過了，可以進 UAT 了。」→ 品管通過，可以請客人試吃了
- **客戶：**「UAT 的時候發現這個流程不順，能不能改？」→ 試用時發現問題

## 為什麼要知道這個詞？

UAT 是上線前的最後一道關卡。如果你是 PM 或業務，你很可能就是 UAT 的執行者——你要實際操作系統，確認「這就是我們要的東西」。通過 UAT，才能正式[部署](deploy.md)上線。
