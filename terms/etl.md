# ETL — 搬家公司的三步驟

> **白話說：** 把資料「抽出來（Extract）、整理好（Transform）、放進去（Load）」的自動化流程。

---

## 它到底是什麼？

**ETL** 是三個英文字的縮寫：

| 字母 | 全名 | 白話 | 搬家比喻 |
|------|------|------|---------|
| **E** | Extract | 從來源把資料抽出來 | 搬家公司去舊家搬東西出來 |
| **T** | Transform | 清理、整理、轉換格式 | 打包分類：衣服一箱、書一箱、易碎品一箱 |
| **L** | Load | 把整理好的資料放到目的地 | 搬到新家，一箱一箱歸位 |

```
舊家（資料來源）→ 搬出 → 打包整理 → 搬入新家（目的地）
```

**ETL 就是搬家公司——負責把東西搬出、打包好、搬到新家歸位。**

## 生活比喻 / 實際例子

舉個實際的例子——你的公司有三個系統：官網、App、POS 機。每個系統都有自己的銷售資料。老闆想看「所有通路加起來」的報表：

```
E（Extract）：從官網、App、POS 三個地方各抽一份銷售資料
T（Transform）：把三份格式不同的資料統一格式、去掉重複、算好小計
L（Load）：把整理好的資料匯入報表系統
```

### 常見情境

- **資料工程師：**「ETL 排程昨晚跑失敗了，今天的報表數字不對。」→ 搬家公司昨晚沒來，新家空的
- **PM：**「這個報表的資料來源是什麼？」→ 「ETL 從訂單系統和會員系統各抽了一份。」
- **老闆：**「為什麼報表的數字跟後台不一樣？」→ 可能是 ETL 的 Transform 步驟有問題

## 為什麼要知道這個詞？

只要你的公司有做報表、做數據分析、做[儀表板](dashboard.md)，背後幾乎都有 ETL 在跑。當報表數字不對，通常第一步就是去查 ETL 是不是出了問題。
