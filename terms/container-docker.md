# Container / Docker（容器）— 標準化貨櫃

> **白話說：** 把程式和它需要的環境打包在一起，到哪台電腦都能跑，不會出現「在我電腦明明可以跑」的問題。

---

## 它到底是什麼？

**Container**（容器）是一種把程式和環境打包在一起的技術。**Docker** 是目前最常用的容器工具。

想像國際貨運——

```
沒有貨櫃（沒有 Container）：
  每種貨物要用不同的方式搬運，到港口還要重新整理，很容易出錯

有貨櫃（有 Container）：
  不管裡面裝什麼——衣服、電器、食品——都塞進標準化的貨櫃
  → 任何船都能載、任何港口都能卸、任何卡車都能拉
```

**Container 就是標準化貨櫃——不管裡面裝什麼，包好了到哪都能用。**

## 生活比喻 / 實際例子

工程師最怕的一句話就是「在我的電腦可以跑啊」。因為每個人的電腦環境不同——作業系統不同、軟體版本不同、設定不同。Container 就是把「程式 + 它需要的一切環境」打包起來，到哪台電腦打開都一模一樣。

### 常見情境

- **DevOps（負責維運和自動化部署的）工程師：**「把這個服務包成 Docker image（映像檔，就像貨櫃的設計藍圖），[部署](deploy.md)到任何[伺服器](server.md)都能跑。」
- **工程師：**「你 Docker 跑起來了嗎？」→ 你有把開發環境的容器啟動嗎？
- **PM：**「為什麼用 Docker？」→ 「確保每個環境都一樣，不會出現本地能跑、線上掛掉的問題。」

## 為什麼要知道這個詞？

現代軟體幾乎都會用到 Docker / Container。當你聽到工程師說「打包成 image（映像檔）」、「跑個 container」，他們就是在用這種「標準化貨櫃」的概念來確保程式到哪都能正常運作。
