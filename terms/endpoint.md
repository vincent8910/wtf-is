# Endpoint（端點）— 郵局的不同窗口

> **白話說：** 後端開出來讓人取資料的特定地址，不同的 endpoint 負責不同的功能。

---

## 它到底是什麼？

**Endpoint** 就是後端伺服器上一個一個特定的網址。前端或其他系統想要某個功能，就去打對應的 endpoint。

想像郵局有不同的窗口——

```
窗口 1（寄信）    → /api/mail/send
窗口 2（領包裹）  → /api/package/pickup
窗口 3（買郵票）  → /api/stamp/buy
```

**每個窗口（endpoint）負責不同的服務。** 你不會跑去寄信窗口領包裹。同樣地，你不會用「查會員資料」的 endpoint 去做「下訂單」的事。

## 生活比喻 / 實際例子

就像一棟大樓的不同樓層——1 樓是大廳、2 樓是餐廳、3 樓是會議室。你想開會就去 3 樓，想吃飯就去 2 樓。Endpoint 就是告訴你「去哪裡拿你要的東西」。

### 常見情境

- **前端工程師：**「這個 endpoint 回傳的資料格式是什麼？」→ 我去這個地址拿資料，你給我什麼格式？
- **後端工程師：**「我開好三個 endpoint 了，文件在這裡。」→ 我準備好三個窗口了
- **PM：**「這個功能需要新開一個 endpoint 嗎？」→ 需要再多開一個服務窗口嗎？

## 為什麼要知道這個詞？

如果你在看 [API](api.md) 文件或跟工程師討論功能，他們常常會說「打這個 endpoint」。知道這個詞，你就知道他們在說「去某個特定的地址取得或送出資料」。
