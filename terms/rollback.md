# Rollback（回滾）— 按下 Ctrl+Z 回到上一步

> **白話說：** Rollback 就像按 Ctrl+Z（復原鍵）——把系統回復到上一個正常的版本。

---

## 它到底是什麼？

[部署](deploy.md) 了新版本但出問題時，最快的止損方式就是 **Rollback**——把系統「倒回去」跑舊版本。

```
遊戲打魔王打不過 → 讀取上一個存檔重來
新版本上線出問題 → Rollback 到上一個穩定版本
```

不用花時間在線上 debug，先退回安全狀態，等穩定了再慢慢修。

## 為什麼 Rollback 很重要？

| 沒有 Rollback | 有 Rollback |
|--------------|------------|
| 新版壞了，只能硬撐著在線上修 | 一鍵退回舊版，客戶幾乎沒感覺 |
| 修的時候可能越改越壞 | 先穩定再說，從容找原因 |
| 客戶持續受影響 | 影響時間降到最短 |

## 日常場景

- **「新版本有問題，先 rollback」**
  → 退回上一個正常版本，別讓客戶繼續受影響
- **「我們有 rollback 的方案嗎？」**
  → 在部署前先確認：如果出事了，能不能快速退回？
- **「rollback 之後再查原因」**
  → 先止損，再慢慢找問題

> 💡 [Git](git.md) 本身就是 rollback 的好工具——它記住每一次的版本，你隨時可以回到過去。
