# UI / UX — 餐廳裝潢 vs 用餐體驗

> **白話說：** UI 是畫面長什麼樣（好不好看），UX 是用起來順不順（好不好用）。

---

## 它到底是什麼？

這兩個詞常常被放在一起講，但其實是兩件不同的事：

- **UI（User Interface，使用者介面）**：你眼睛看到的一切——按鈕長什麼樣、顏色配得好不好看、字體大不大
- **UX（User Experience，使用者體驗）**：你操作起來的感受——找不找得到功能、流程順不順、會不會迷路

```
UI 好、UX 差：餐廳裝潢超美，但菜單找不到、服務員態度差、等餐等很久
UI 差、UX 好：路邊攤長得普通，但點餐快、東西好吃、老闆記得你的口味
UI 好、UX 好：漂亮的餐廳 + 流暢的用餐體驗 = 完美
```

## 生活比喻 / 實際例子

**UI 是餐廳裝潢，UX 是用餐體驗。**

你可能去過一家裝潢很漂亮的餐廳，結果動線很差、菜單看不懂、結帳要等很久。那就是「UI 好但 UX 差」。

反過來，巷口的滷肉飯攤子可能沒什麼裝潢，但你一坐下老闆就知道你要什麼、五分鐘上菜、好吃又便宜。那就是「UX 很好」。

### 常見情境

- **老闆：**「我們的 App 看起來很漂亮啊，為什麼用戶還是一直客訴？」→ UI 可能不錯，但 UX 需要改善
- **設計師：**「這個按鈕的 UI 沒問題，但從 UX 角度來看，放在這裡用戶會找不到。」
- **PM：**「我們需要做 UX 研究，看看用戶卡在哪一步。」

## 為什麼要知道這個詞？

因為你每天都在當「使用者」。當你說「這個 App 好難用」，你其實在講 UX；當你說「這個網站好醜」，你在講 UI。分清楚這兩個詞，跟設計師和工程師溝通時會更精準。
