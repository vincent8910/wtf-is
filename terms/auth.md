# Authentication / Authorization — 驗明正身 vs 權限管理

> **白話說：** Authentication 是門口警衛確認「你是誰」，Authorization 是確認「你能去哪」。

---

## 它到底是什麼？

這兩個詞連工程師都常搞混！用公司大樓的比喻：

```
你走進公司大樓
  │
  ├─ 🪪 Authentication（驗證）
  │   門口警衛看你的員工證：「你是誰？」
  │   → 確認你是不是這棟大樓的人
  │
  └─ 🔐 Authorization（授權）
      你要進機房，但門禁卡刷不過：「你有權限嗎？」
      → 你是員工沒錯，但不是每個地方你都能進
```

## 一張表搞懂

| | Authentication（驗證） | Authorization（授權） |
|---|---|---|
| **在問什麼** | 你是誰？ | 你能做什麼？ |
| **比喻** | 門口刷門禁卡進大樓 | 特定樓層才能刷進去 |
| **常見方式** | 帳號密碼、指紋、人臉辨識 | 角色權限（管理員/一般用戶） |
| **失敗的結果** | 「登入失敗」 | 「你沒有權限檢視此頁面」 |
| **常見縮寫** | AuthN | AuthZ |

### 工程師說的 "Auth" 到底是哪個？

**Auth** 可能指兩個中的任何一個，要看上下文：
- 「auth 做好了嗎？」→ 通常指整個登入驗證系統
- 「auth 設定有問題」→ 要再問清楚是驗證還是授權

## 日常場景

- **「使用者登入是 authentication」**
  → 確認你是誰
- **「能不能看某個頁面是 authorization」**
  → 確認你有沒有權限
- **「這個 API 需要 auth」**
  → 呼叫這個功能前要先證明你的身分

> 💡 最簡單的記法：Authentication = 「**你是不是**你」，Authorization = 「**你能不能**做這件事」。
