# Component（元件）— 樂高積木

> **白話說：** 畫面上可以重複使用的小零件，組合起來就變成一個完整的頁面。

---

## 它到底是什麼？

**Component** 就是把網頁畫面拆成一塊一塊可以重複使用的小零件。

想像你在玩樂高——

```
一個按鈕 = 一塊積木
一個搜尋欄 = 一塊積木
一張產品卡片 = 一塊積木
整個首頁 = 很多積木拼在一起
```

**每塊樂高積木都可以獨立使用，也可以在不同地方重複使用。** 你不用每次都從頭做，做好一個按鈕元件，整個網站的按鈕都能共用。

## 生活比喻 / 實際例子

就像便利商店的貨架模組——每一格貨架的結構都一樣，只是裡面放的商品不同。你不用幫每個商品客製一個貨架，同一種貨架重複使用就好。

### 常見情境

- **設計師：**「這個卡片 component 在首頁和搜尋頁都會用到，樣式要統一。」
- **前端工程師：**「我先做好按鈕的 component，其他頁面直接引用就好。」
- **PM：**「改了這個 component，是不是所有用到它的地方都會一起改？」→ 對，這就是元件化的好處

## 為什麼要知道這個詞？

當設計師畫好 [UI](ui-ux.md)（畫面），工程師會把畫面拆成一個個 component 來開發。如果你在開會時聽到「這個 component 可以共用」、「我們先把 component library（元件庫，把做好的零件統一收好）建起來」，知道他們在說的就是「可重複使用的畫面零件」就好。
