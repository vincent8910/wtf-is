# API — 軟體之間溝通的窗口

> **白話說：** API 就像餐廳的**服務窗口**——你（軟體 A）把點餐單遞進去，廚房（軟體 B）做好了再從窗口端出來。你不用進廚房，廚房也不用出來找你。

---

## 它到底是什麼？

**API** 是 **Application Programming Interface** 的縮寫，中文叫「應用程式介面」。

聽起來很硬，但概念超簡單：

> **API 就是「兩個軟體互相講話的規則」。**

就像你去麥當勞：

```
你（顧客）      →  櫃台（API）  →  廚房（系統）
「我要一份大麥克」  →  點餐機接收  →  廚房開始做
                  ←  取餐叫號   ←  做好了
```

你不用知道廚房怎麼運作，廚房也不用管你是誰。**櫃台就是 API**——它定義了「你可以點什麼」以及「怎麼拿到結果」。

## 生活中到處都是 API

你每天都在「用 API」，只是不知道：

| 你做的事 | 背後的 API |
|---------|-----------|
| 用 Google 地圖查路線 | 你的手機透過 API 問 Google 伺服器要路線資料 |
| 用 LINE Pay 付款 | LINE 透過 API 跟銀行確認扣款 |
| 看天氣 App | App 透過 API 跟氣象局拿最新天氣 |
| 用 Claude Code | Claude Code 透過 API 把你的問題傳給 Anthropic 的 AI |

## 跟 API Key 的關係

[API](api.md) 是那個「服務窗口」，而 [API Key](api-key.md) 是你的「會員卡」。

```
你想用 Anthropic 的 AI 服務
├─ API = 服務窗口（你從這裡點餐）
├─ API Key = 會員卡（證明你有資格點餐）
└─ 結果 = AI 的回答（你點的餐做好了）
```

沒有 API，你連窗口都找不到。
沒有 API Key，你站在窗口前但店員不理你。

## 為什麼要知道這個詞？

因為你會在很多地方碰到它：

- 「Claude Code 是透過 **API** 連到 Anthropic 的 AI」
- 「這個服務有開放 **API**，可以讓 AI 去串接」
- 「[MCP](mcp.md) 就是讓 AI 透過 **API** 連接各種外部工具」

搞懂 API，你就能理解為什麼 AI 工具能「連上」這麼多服務——因為到處都有窗口可以對接。
