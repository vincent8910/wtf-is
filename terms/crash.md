# Crash（閃退）— 自動門壞了把你彈出去

> **白話說：** App 突然當掉，被踢回手機桌面，什麼都來不及存。

---

## 它到底是什麼？

**Crash** 就是 App 在執行中突然崩潰、強制關閉。你正在用得好好的，畫面突然消失，回到手機桌面。

想像你走進一家自動門——

```
正常情況：自動門偵測到你 → 門打開 → 你走進去 → 門關上
Crash：自動門偵測到你 → 門打開一半 → 門突然壞掉 → 砰！把你彈出去
```

**Crash 就是自動門故障，毫無預警地把你彈出去。**

## 生活比喻 / 實際例子

你有沒有在用某個 App 的時候，畫面突然黑一下就回到手機桌面了？那就是 crash。

### 常見的 crash 原因

- 程式碼有 [bug](bug-debug.md)，遇到預料之外的情況就崩潰
- 手機記憶體不夠，系統強制關掉你的 App
- 新版本更新後，舊的資料格式不相容

### 常見情境

- **用戶客訴：**「你們的 App 一打開就閃退！」→ 一打開就 crash
- **Mobile 工程師：**「我查了 crash [log](log.md)（崩潰紀錄），是空值（程式預期要有資料，結果什麼都沒有）沒處理到。」→ 崩潰紀錄顯示程式遇到了沒預料到的狀況
- **PM：**「這個版本的 crash rate 是多少？」→ 每 100 次使用中有幾次閃退？
- **QA（品質測試人員）：**「我操作到第三步就 crash 了。」→ 做到第三步 App 就閃退

## 為什麼要知道這個詞？

Crash 是影響用戶體驗最嚴重的問題之一。如果 App 動不動就閃退，用戶會直接給一星負評然後刪掉。所以「crash rate」（閃退率）是所有 App 團隊密切關注的指標。
