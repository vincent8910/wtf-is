# Native App / Web App — 實體店面 vs 快閃店

> **白話說：** Native App 是從商店下載安裝的 App，Web App 是用瀏覽器直接開的網頁版應用。

---

## 它到底是什麼？

這兩種是做手機應用程式的不同方式：

- **Native App**：從 App Store 或 Google Play 下載、安裝到手機裡的 App
- **Web App**：不用下載，用手機瀏覽器打開就能用的網頁版應用

```
Native App（實體店面）：
  → 你要特地走去那家店（下載安裝）
  → 裝潢精緻、體驗好、什麼功能都有
  → 但換地址（更新版本）要重新公告

Web App（快閃店）：
  → 路過就能逛（打開瀏覽器就能用）
  → 設備比較簡單、體驗差一點
  → 但隨時可以換內容、不用等客人「更新」
```

## 生活比喻 / 實際例子

| 比較 | Native App | Web App |
|------|-----------|---------|
| 怎麼用 | 從商店下載安裝 | 瀏覽器輸入網址 |
| 速度 | 比較快、比較順 | 看網路速度 |
| 功能 | 可以用相機、推播、離線 | 功能受限 |
| 更新 | 要去商店更新 | 打開就是最新版 |
| 例子 | Line、Instagram | 手機版 Gmail 網頁 |

### 常見情境

- **老闆：**「我們的 App 一定要讓人下載嗎？」→ 你在問要做 Native 還是 Web App
- **Mobile 工程師：**「做 Native 的話 iOS 和 Android 要各寫一份。」→ 兩個平台的程式碼不一樣
- **PM：**「先做 Web App 快速驗證，反應好再做 Native。」→ 先開快閃店，生意好再開實體店

## 為什麼要知道這個詞？

每次要開發手機應用，第一個要決定的就是「做 Native 還是 Web App」。這決定了開發時間、成本、使用者體驗、上架流程。知道這兩者的差別，你就能在討論中做出更好的判斷。
