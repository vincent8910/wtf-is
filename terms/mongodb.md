# MongoDB — 用 JSON 格式存資料的資料庫

> **白話說：** 如果傳統[資料庫](database.md)是 Excel 表格（一格一格整齊排好），MongoDB 就是一個超大的檔案櫃，每份文件格式可以不一樣。

---

## 它到底是什麼？

MongoDB 是一種「NoSQL 資料庫」。傳統資料庫（如 MySQL）用表格存資料，像 Excel 一樣每一行的欄位都要一致。MongoDB 不一樣——它用 [JSON](json.md) 格式存資料，每筆資料可以有不同的欄位。

**傳統資料庫（MySQL）像 Excel：**
```
| 姓名 | 年齡 | Email |
|------|------|-------|
| 小明 | 25   | a@b.c |
| 小美 | 30   | d@e.f |
```
每一行的欄位必須一模一樣。

**MongoDB 像檔案櫃：**
```
{ "姓名": "小明", "年齡": 25, "Email": "a@b.c" }
{ "姓名": "小美", "年齡": 30, "興趣": ["畫畫", "旅行"] }
```
每筆資料的欄位可以不一樣，格式彈性很大。

## 生活比喻 / 實際例子

- **傳統資料庫（MySQL）** = 圖書館的書目卡片，每張卡片格式統一：書名、作者、出版社、ISBN
- **MongoDB** = 一個大書架，每本書可以是不同大小、不同格式（有的是漫畫、有的是小說、有的是雜誌），只要你找得到就好

實際例子：
- 工程師說：「這個專案的 DB 用 **MongoDB**，資料格式變動比較彈性」
- 工程師說：「用戶資料每個人的欄位都不太一樣，用 **MongoDB** 比較方便」

## 為什麼要知道這個詞？

- 很多新創和中小型專案會選 MongoDB，因為開發速度快、格式彈性
- 當工程師說「我們用 NoSQL」或「用 Mongo」，就是在用這種資料庫
- MongoDB 跟 [Redis](redis.md) 常搭配使用：MongoDB 存主要資料，Redis 存[快取](cache.md)
