# Token（令牌） — 系統發給你的臨時通行證

> **白話說：** 就像遊樂園入場後手上蓋的印章，憑這個印章在裡面玩什麼設施都不用重新買票。

---

## 它到底是什麼？

當你登入一個網站或 App，系統確認「你是你」之後，會發給你一個 **Token**——一串很長的亂碼文字。之後你在這個網站上做任何操作，都是靠這個 token 來證明「我已經登入過了」，不用每次都重新輸入帳號密碼。

Token 通常有「期限」，過期了就要重新登入。這也是為什麼你有時候隔幾天再開 App 會被要求重新登入。

## 生活比喻 / 實際例子

- **手上蓋的入場章**：進遊樂園驗票後手上蓋章，之後在裡面玩設施出示手上的章就好
- **飯店房卡**：Check-in 之後拿到房卡，之後用房卡就能進房間、用健身房、搭電梯，退房後房卡就失效
- **星巴克隨行卡的 QR Code**：你已經加值了（登入），之後用 QR Code 就能結帳（操作），不用每次重新加值

### Token 跟密碼有什麼不同？

| | 密碼 | Token |
|--|------|-------|
| 誰建的 | 你自己設定的 | 系統自動產生的 |
| 長什麼樣 | 你記得住的文字 | 一長串亂碼 |
| 有效期 | 你不改就一直有效 | 通常有時間限制 |
| 用途 | 拿來「登入」 | 登入後拿來「證明身分」 |

## 為什麼要知道這個詞？

- 「Token 過期了」→ 你的臨時通行證到期了，要重新登入
- 「把 token 帶在 header 裡」→ 工程師在說每次呼叫 [API](api.md) 時要附上通行證
- 「Token 不要存在前端」→ 通行證不要放在容易被偷的地方（資安考量）
- 「refresh token」→ 用來換新通行證的「換證憑據」，讓你不用一直重新登入
