# Clone — 把別人的專案完整複製一份到你的電腦

> **白話說：** 就像在 Google Drive 上按「建立副本」，把整份文件複製一份到你自己的電腦裡。

---

## 它到底是什麼？

Clone 就是把一個 [Repo](repo.md)（專案資料夾）從遠端（通常是 [GitHub](github.md)）**完整複製一份到你的電腦上**。

複製下來的不只是檔案本身，還包括這個專案的**所有修改歷史紀錄**。所以你拿到的不是某一個時間點的快照，而是從頭到尾的完整備份。

Clone 跟一般的「下載」有什麼不同？

- **一般下載**：你只拿到檔案，跟原本的專案就斷開了
- **Clone**：你拿到檔案＋完整歷史，而且跟遠端的 repo 還保持連線，之後可以繼續同步更新

## 生活比喻 / 實際例子

想像公司的共用雲端硬碟上有一份提案企劃：

- 你按「建立副本」把整份企劃複製到你的電腦 → 這就是 **clone**
- 你在自己的電腦上修改、加東西 → 這是在 local repo 工作
- 改好之後，你再把修改同步回雲端硬碟 → 這就是 [push](push.md)

工程師會說：
- 「你先把 **repo clone** 下來，然後跑跑看。」 → 先把專案複製到你的電腦
- 「新人第一天就是先 **clone** 專案。」 → 把公司的程式碼下載到自己電腦
- 「我 **clone** 了你的 repo 來研究。」 → 我複製了一份你的專案來看

## 為什麼要知道這個詞？

因為幾乎每個工程師加入新專案的第一步就是 clone。你會在這些場景聽到：

- 新人報到：「先把 repo **clone** 下來。」
- 交接工作：「你 **clone** 這個 repo，裡面有所有文件。」
- 協作開發：「大家各自 **clone** 一份，改好再發 [PR](pull-request.md)。」

簡單記：**Clone = 把雲端的專案完整複製到我的電腦上**。
