# Open Source（開源） — 程式碼公開、大家都能看、都能用、都能改

> **白話說：** 就像食譜公開的餐廳——任何人都能看到配方，也能拿去改良開自己的店。

---

## 它到底是什麼？

一般公司寫的軟體，程式碼都是「關起門來寫」的，外人看不到裡面怎麼做的。

但 **Open Source（開源）** 專案不一樣：它把程式碼完全公開，放在 [GitHub](github.md) 這類平台上，任何人都可以：

- **看**：研究它是怎麼寫的
- **用**：免費拿去用在自己的專案
- **改**：覺得哪裡可以更好，自己改完後提交回去
- **分享**：把自己改過的版本分享出去

## 生活比喻 / 實際例子

- **公開食譜**：不只告訴你菜名，還把整份食譜公開，你可以自己做、也可以加自己的調味
- **維基百科**：任何人都可以編輯、修正、補充內容
- **社區公園**：大家共同維護，誰都能來用，有人覺得哪裡可以更好就動手改善

## 為什麼要知道這個詞？

- 「這個工具是 open source 的」→ 免費、公開、可以放心用（但要注意授權條款）
- 「我們用的是開源方案」→ 團隊選了一個公開的工具，而不是花錢買商業軟體
- 「去 GitHub 上提一個 issue」→ 在開源專案的公開平台上回報問題或提建議
- 「這個 library 的 license 是什麼？」→ 在確認這個開源工具的使用規則（有些開源工具有商業使用限制）
