# Data Lake / Data Warehouse — 儲藏室 vs 圖書館

> **白話說：** Data Lake 是什麼資料都先丟進去，Data Warehouse 是整理分類好的資料庫。

---

## 它到底是什麼？

這兩個都是存放大量資料的地方，但整理程度不同：

- **Data Lake（資料湖）**：什麼都先丟進來，不管格式、不管有沒有用，之後再說
- **Data Warehouse（資料倉儲）**：只放整理好、分類好、格式統一的資料

```
Data Lake（儲藏室）：
  → 紙箱、舊書、雜物、照片、發票⋯⋯全丟進去
  → 很方便，但要找東西要翻很久

Data Warehouse（圖書館）：
  → 書分門別類、貼好標籤、排列整齊
  → 要找什麼一查就有，但上架前要先整理
```

## 生活比喻 / 實際例子

**Data Lake 是你家的儲藏室，Data Warehouse 是圖書館。**

儲藏室什麼都丟，方便但亂；圖書館整整齊齊，但每本書要先分類編碼才能上架。

### 實際的流程

通常資料會先進 Data Lake，再經過 [ETL](etl.md) 整理後放進 Data Warehouse：

```
各種來源的原始資料 → 丟進 Data Lake（先收著）
                    → ETL 整理轉換
                    → 放進 Data Warehouse（整理好了）
                    → Dashboard 報表用
```

### 常見情境

- **資料工程師：**「原始 [log](log.md)（系統紀錄）先丟進 data lake，之後再清洗。」→ 先丟進儲藏室，之後再整理
- **PM：**「這個報表的資料從哪裡來？」→ 「從 data warehouse 拉的，已經是整理好的資料。」
- **老闆：**「我們有沒有保留用戶的行為資料？」→ 「有，都在 data lake 裡，但還沒整理。」

## 為什麼要知道這個詞？

當你需要做數據分析或報表，你會想知道資料是從哪裡來的。Data Lake 裡的資料「原始但完整」，Data Warehouse 裡的資料「整理好可以直接用」。搞清楚這兩個的差別，你就知道為什麼有些資料可以馬上查到，有些要等工程師整理。
