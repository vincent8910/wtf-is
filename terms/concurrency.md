# Concurrency（並發）— 跨年夜的 7-11 店員

> **白話說：** 伺服器同時處理很多人的請求，不讓任何人等太久。

---

## 它到底是什麼？

**Concurrency**（並發）就是系統同時處理多個任務的能力。

想像跨年夜的 7-11——

```
只有一個店員，一次只服務一個客人：
  客人 A 結帳 → 客人 B 等 → 客人 C 等 → 排到明年

一個超強店員，同時處理很多事：
  一邊幫 A 結帳 → 一邊跟 B 說微波爐左邊 → 一邊幫 C 加值悠遊卡
```

**Concurrency 就是那個跨年夜的 7-11 店員，同時應付一堆客人還不崩潰。**

## 生活比喻 / 實際例子

也像餐廳的廚師——他不會煮完一道菜才開始煮下一道。而是同時燒水、切菜、炒鍋上的菜，交錯處理多道料理，讓所有客人都能在合理時間內上菜。

### 常見情境

- **遊戲伺服器工程師：**「這個伺服器要能處理 1 萬人同時在線的 concurrency。」
- **PM：**「雙 11 當天預期多少 concurrent users（同時在線用戶）？」
- **工程師：**「concurrency 沒處理好，一堆人同時搶會出 [bug](bug-debug.md)（程式錯誤）。」→ 例如兩個人同時搶到最後一張票

## 為什麼要知道這個詞？

每次你聽到「這個系統撐不住」、「伺服器被擠爆」，背後通常就是 concurrency 的問題——同時進來的人太多，系統處理不過來。在討論系統容量、活動規劃（搶票、特賣）時，這是很重要的詞。
