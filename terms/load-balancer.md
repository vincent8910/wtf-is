# Load Balancer（負載均衡器）— 百貨公司手扶梯引導員

> **白話說：** 把大量使用者的請求分散到不同伺服器，不讓某一台被擠爆。

---

## 它到底是什麼？

**Load Balancer** 就是站在伺服器前面的「交通指揮」，負責把進來的請求分配到不同的伺服器去處理。

想像百貨公司週年慶——

```
沒有 Load Balancer：
  只開一個門 → 所有人擠同一個入口 → 門口塞爆、有人進不去

有 Load Balancer（引導員）：
  「A~H 請走左邊！」「I~P 請走中間！」「Q~Z 請走右邊！」
  → 人潮分散到三個入口 → 大家都能順利進場
```

**Load Balancer 就是百貨公司門口的引導員——把人潮分散到不同入口，避免某個入口被擠爆。**

## 生活比喻 / 實際例子

也像高速公路的收費站——如果只有一個車道，大家都塞在那裡。多開幾個車道（多台伺服器），再有人在前面引導分流（Load Balancer），車流就順暢了。

### 常見情境

- **DevOps（負責維運和自動化部署的）工程師：**「我在前面加了一個 load balancer，流量（同時進來的使用者請求數量）會自動分到三台 server。」
- **PM：**「雙 11 撐得住嗎？」→ 「load balancer 會自動分流，應該沒問題。」
- **工程師：**「有一台 server 掛了，但 load balancer 自動把流量導到其他台，用戶沒感覺。」

## 為什麼要知道這個詞？

當你的服務使用者越來越多，一台[伺服器](server.md)不夠用了，就需要多台一起扛。Load Balancer 就是讓多台伺服器協作的關鍵角色。每次討論「系統能不能承受大流量」，Load Balancer 一定會被提到。
