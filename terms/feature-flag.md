# Feature Flag（功能開關） — 不用重新部署，就能開關某個功能

> **白話說：** 就像電燈開關——功能已經裝好了，但你可以隨時決定要不要打開讓使用者看到。

---

## 它到底是什麼？

Feature Flag 就是在程式碼裡加一個「開關」。功能已經寫好、也已經[部署](deploy.md)上線了，但可以透過設定來控制：

- **開關打開** → 使用者看得到這個新功能
- **開關關掉** → 使用者完全不知道這個功能存在

這樣就算新功能有問題，也不用緊急改程式碼再重新部署，只要把開關關掉就好。

## 生活比喻 / 實際例子

想像你開了一家餐廳，新菜色已經準備好了：

- **沒有 Feature Flag** = 新菜直接印在菜單上，如果客人吃了不喜歡，你得重印菜單
- **有 Feature Flag** = 新菜寫在一張活頁紙上夾在菜單裡，不好吃就抽掉，不用重印

更進階的玩法：
- 只讓 10% 的使用者看到新功能（灰度發佈）
- 只讓內部員工先試用
- 只讓台灣地區的使用者看到

實際例子：
- PM 說：「新的購物車功能先用 **feature flag** 控制，只開給內部測試」
- 工程師說：「先把 **feature flag** 關掉，等 [bug](bug-debug.md) 修好再開」

## 為什麼要知道這個詞？

- PM 在規劃功能上線策略時常會用到 feature flag
- 它讓「部署」和「上線」變成兩件事：功能可以先部署但不開放給使用者
- 出問題時可以秒關功能，不用等工程師重新部署，大幅降低風險
