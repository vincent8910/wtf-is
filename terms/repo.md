# Repo（Repository） — 放專案所有檔案的資料夾

> **白話說：** 就像一個專案的公事包，裡面裝了所有文件、紀錄和修改歷史。

---

## 它到底是什麼？

Repo 是 Repository（倉庫）的縮寫。簡單來說，它就是一個**存放整個專案所有東西的地方**——程式碼、文件、圖片、設定檔，加上每一次修改的歷史紀錄，全部都在裡面。

你可以把它想像成一個超級資料夾：不只存檔案，還會記住「誰、在什麼時候、改了什麼」。

Repo 通常有兩種：

- **Local repo（本地）**：在你自己電腦上的那份
- **Remote repo（遠端）**：放在 [GitHub](github.md) 上的那份，大家都能看到、一起協作

## 生活比喻 / 實際例子

想像你在做一個提案企劃：

- 你的電腦裡有一個資料夾，裡面有簡報、文案、素材 → 這是你的 **local repo**
- 你把它上傳到公司的共用雲端硬碟，讓同事也能看到、一起改 → 這是 **remote repo**
- 雲端硬碟還會記錄「誰在幾點改了哪個檔案」 → 這就是 repo 的版本紀錄功能

工程師會說：
- 「你去把 **repo** clone 下來。」 → 把專案從雲端下載一份到你的電腦
- 「這個 **repo** 在 GitHub 上是公開的。」 → 任何人都能看到這個專案的內容
- 「我開了一個新 **repo**。」 → 我建立了一個新專案的資料夾（包含版本控制）

## 為什麼要知道這個詞？

因為工程師幾乎每天都在講這個詞。只要涉及專案、程式碼、協作，repo 就一定會出現：

- PM 問進度：「東西在哪個 **repo**？」
- 工程師交接：「我把 **repo** 權限開給你。」
- 新人報到：「先把 **repo** clone 下來熟悉一下。」

搞懂 repo，你就知道大家口中的「專案放在哪裡」到底是什麼意思。
