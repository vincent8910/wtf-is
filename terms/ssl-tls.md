# SSL / TLS（安全憑證） — 讓網站連線加密的安全鎖

> **白話說：** 就像在你跟網站之間架一條加密隧道，別人看得到你們在通訊，但看不懂內容。

---

## 它到底是什麼？

SSL（Secure Sockets Layer）和 TLS（Transport Layer Security）是讓網站連線加密的技術。TLS 是 SSL 的升級版，但大家習慣還是叫「SSL 憑證」。

你有沒有注意過，有些網站的網址列會出現一個小鎖頭圖示 🔒？那就代表這個網站有裝 SSL/TLS 憑證，你跟它之間的通訊是加密的。

- 有鎖頭 = `https://`（安全）
- 沒有鎖頭 = `http://`（不安全）

如果沒有 SSL/TLS，你在網站上輸入的帳號密碼、信用卡號碼，就像寫在明信片上一樣，沿途任何人都可以偷看。

## 生活比喻 / 實際例子

想像你寄一封信：

- **沒有 SSL/TLS** = 寄明信片，郵差和路上任何人都能看到內容
- **有 SSL/TLS** = 信放進保險箱再寄出去，只有收件人有鑰匙能打開

實際例子：
- 工程師說：「**SSL 憑證**到期了，要趕快更新，不然瀏覽器會跳警告」
- PM 問：「為什麼使用者看到『此網站不安全』的警告？」工程師：「**SSL 憑證**過期了」
- 老闆說：「我們網站安全嗎？」工程師：「有裝 **SSL**，所有連線都有[加密](encryption.md)」

## 為什麼要知道這個詞？

- 網站上線前一定要裝 SSL 憑證，不然瀏覽器會嚇跑使用者
- Google 會優先顯示有 SSL 的網站，所以也跟 [SEO](seo.md) 有關
- 如果你的網站有會員登入或金流功能，沒有 SSL 等於門戶大開
- SSL 憑證有免費的（Let's Encrypt）也有付費的，通常工程師或 [DevOps](devops.md) 會處理
