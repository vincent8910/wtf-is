# Pipeline（資料管線）— 自來水管線

> **白話說：** 資料自動經過一連串處理步驟，從原始資料變成你能用的成品。

---

## 它到底是什麼？

**Pipeline** 就是資料從 A 點到 B 點之間，自動經過的一連串處理步驟。

想像自來水的處理過程——

```
水庫的原水
  → 沉澱池（去除雜質）
  → 過濾池（過濾細小雜質）
  → 消毒池（加氯消毒）
  → 清水池（儲存乾淨的水）
  → 水管送到你家
```

**Pipeline 就是自來水管線——原水經過一連串處理步驟，最後變成你能喝的乾淨水。**

## 生活比喻 / 實際例子

也像工廠的生產線——原料進來，經過切割、組裝、品檢、包裝，最後變成成品出貨。每一站做一件事，做完自動傳到下一站。

### 在軟體世界的 Pipeline

- **資料 Pipeline**：原始資料 → 清洗 → 轉換 → 存入資料庫 → 生成報表
- **CI/CD Pipeline**：程式碼 → 編譯 → 測試 → 部署 → 上線
- **機器學習 Pipeline**：收集資料 → 清理 → 訓練模型 → 驗證 → 部署

### 常見情境

- **資料工程師：**「pipeline 卡在第三步，資料轉換失敗了。」→ 生產線第三站出了問題
- **PM：**「這個 pipeline 每天自動跑一次嗎？」→ 這條生產線每天自動啟動一次嗎？
- **工程師：**「我在 pipeline 中間加一個驗證步驟。」→ 在生產線中間加一個品檢站

## 為什麼要知道這個詞？

只要你的工作跟數據、報表、自動化流程有關，就會聽到 pipeline。它描述的是「資料自動經過一連串步驟處理」的概念。當有人說「pipeline 壞了」，就是自動化流程的某個步驟出了問題。
