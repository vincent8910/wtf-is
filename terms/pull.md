# Pull（拉取）— 把雲端最新的存檔下載到你的電腦

> **白話說：** [Push](push.md) 是「上傳」，Pull 就是「下載」。把別人的最新進度同步到你的電腦上。

---

## 它到底是什麼？

Pull 是 [Git](git.md) 的操作之一，意思是**從遠端（通常是 [GitHub](github.md)）把最新的程式碼拉回你的電腦**。

想像團隊裡有五個人同時在做一個專案：
1. 小王改了登入頁面，[push](push.md) 上去了
2. 你想要拿到小王的最新版本 → 你就 **pull** 一下

Pull 就是讓你的電腦跟雲端上的版本「同步」，確保你手上的是最新的。

## 生活比喻 / 實際例子

**共用的 Google Drive 資料夾：**
- 你的同事上傳了新版的企劃書到共用資料夾
- 你點「同步」讓你的電腦也拿到最新版 → 這就是 **Pull**

**你什麼時候會聽到：**
- 「你先 **pull** 一下，我剛改了一些東西。」
- 「記得每天開始工作前先 **pull** 最新的 code。」
- 「你的版本太舊了，**pull** 下來看看。」

## 為什麼要知道這個詞？

- Push 和 Pull 是一對的：Push 是上傳你的修改，Pull 是下載別人的修改
- 如果不常 pull，你手上的版本會越來越舊，之後要 [merge](merge.md) 的時候容易出現[衝突](merge-conflict.md)
- 每天開始工作前先 pull 最新版本是好習慣
- 當工程師說「pull 下來跑跑看」，就是「把最新的程式碼載下來，在你電腦上測試」
