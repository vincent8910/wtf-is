# Hotfix — 先貼 OK 繃止血

> **白話說：** Hotfix 就像受傷時先貼 OK 繃——不是根治，但先止血再說。

---

## 它到底是什麼？

**Hot（緊急）+ Fix（修復）= 緊急修復。**

系統在線上出問題——客戶在罵、訂單出錯、錢在燒。來不及走完整流程，先用最快方式修好最痛的部分，這就是 **Hotfix**。

| | 一般修 Bug | Hotfix |
|---|---|---|
| **急迫性** | 排進進度慢慢修 | 現在、立刻、馬上 |
| **流程** | 完整開發流程 | 精簡流程，快速上線 |
| **修復範圍** | 徹底解決 | 先止血，完整修復之後排 |
| **比喻** | 預約家庭醫師 | 衝急診室 |

## 日常場景

- **「先出一版 hotfix 擋一下，完整修復下週再排」**
  → 先止血，根本原因之後處理
- **「這個 hotfix 今天一定要上」**
  → 必須今天 [部署](deploy.md) 到正式環境
- **「不要在 hotfix 上加需求」**
  → 工程師在急診室止血，不要叫他「順便割雙眼皮」

> 💡 Hotfix 不等於「修好了」。OK 繃不是根治，後面通常還需要一次完整修復。
