# Frontend / Backend — 餐廳的外場和內場

> **白話說：** Frontend 是使用者看得到的畫面（外場），Backend 是背後處理資料的系統（內場廚房）。

---

## 它到底是什麼？

你去過餐廳對吧？餐廳分成兩個區域：

- **外場**：你坐的座位、菜單、裝潢、服務生——這些是客人「看得到、摸得到」的
- **內場**：廚房、倉庫、冷凍庫、洗碗機——客人看不到，但沒有它餐廳就完蛋了

軟體世界也一樣：

```
Frontend（前端）= 外場
├── 你看到的網頁畫面
├── 按鈕、輸入框、圖片
├── 動畫、排版、顏色
└── 使用者直接互動的一切

Backend（後端）= 內場
├── 處理登入驗證（你的帳密對不對）
├── 從資料庫撈資料（你的訂單紀錄）
├── 計算金額（折扣、稅金、運費）
└── 跟其他系統溝通（金流、物流）
```

## 一張表看懂差異

| | Frontend（前端） | Backend（後端） |
|---|---|---|
| **比喻** | 餐廳外場 | 餐廳內場（廚房） |
| **誰看得到** | 使用者直接看到 | 使用者看不到 |
| **負責什麼** | 畫面長怎樣、好不好操作 | 資料怎麼存、邏輯怎麼跑 |
| **常用技術** | HTML、CSS、JavaScript、React | Python、Java、Node.js、SQL |
| **出問題的症狀** | 畫面跑版、按鈕沒反應、動畫卡頓 | 資料不見、登入失敗、速度超慢 |

## 日常場景

你一定會在這些對話裡聽到：

- **「這個功能前端就能處理，不用動後端」**
  → 意思是：畫面上改一下就好，不用動到後面的資料處理邏輯
- **「這是前端的 bug 還是後端的？」**
  → 意思是：是畫面顯示出問題，還是背後的資料本身就錯了？
- **「後端 API 還沒好，前端先用假資料做」**
  → 意思是：廚房還沒開始出菜，外場先用模型展示菜色給客人看

## Full-Stack（全端）是什麼？

如果一個工程師同時會做 Frontend 和 Backend，就叫 **Full-Stack（全端）工程師**。

用餐廳來說，就是一個人**既能在外場服務客人，又能進廚房炒菜**——十八般武藝樣樣通。
