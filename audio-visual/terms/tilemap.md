# Tilemap（地圖磚塊）— 用重複的小方塊拼出遊戲地圖

> **白話說：** 就像用樂高底板和不同顏色的積木塊拼出一張地圖。草地用綠色塊、水池用藍色塊、道路用灰色塊，拼一拼就是一個完整的遊戲世界。

---

## 它到底是什麼？

Tilemap 是 2D 遊戲中用來建造地圖和場景的技術。做法是：先準備一組「磚塊」（Tile），每個磚塊是一小張固定大小的圖片（例如 16x16 或 32x32 像素），代表不同的地形或物件——草地、泥土、水面、牆壁、樓梯⋯⋯然後像拼積木一樣，在網格上把這些磚塊一個一個擺放，拼出完整的遊戲地圖。

這個做法的好處是超級省資源。你不需要為每一個關卡畫一整張超大的背景圖，只需要幾十個小磚塊，就能拼出無數種不同的地圖。而且修改很方便——想在這裡加一條河？把幾個草地磚塊換成水面磚塊就好了。

從最早的《超級瑪利歐》到現在的《薩爾達傳說》2D 版，幾乎所有 2D 遊戲的地圖都是用 Tilemap 的概念建造的。就連一些 3D 遊戲的小地圖或 UI 也會用到 Tilemap。

## 生活比喻 / 實際例子

想像你有一盒不同顏色的樂高底板積木：綠色代表草地、藍色代表水、灰色代表石頭路、棕色代表泥土。你拿一塊大底板，然後把這些小積木一格一格擺上去，擺好之後退一步看——哇，一張遊戲地圖就出來了！

要改地圖？把某幾格換個顏色就好。要做新關卡？同一組積木重新排列組合。這就是 Tilemap 的精髓：用少量的素材，創造出大量的內容。

**造句**：「第二關的 **Tilemap** 排好了，但還缺一些裝飾用的磚塊，像是花叢和石頭。」

## 為什麼要知道這個詞？

- 幾乎所有 2D 遊戲都在用 Tilemap，是最基礎的地圖製作方式
- 遊戲企劃在設計關卡時，「畫 Tilemap」是第一步
- 跟美術溝通時，「Tileset（磚塊組）」「Tile 大小」「地圖圖層」是基本用語
- 跟 [Sprite](sprite.md)（精靈圖）和 [Pixel Art](pixel-art.md)（像素畫）是好朋友，經常一起出現
