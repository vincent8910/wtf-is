# Sprite（精靈圖）— 2D 遊戲裡的角色或物件圖片

> **白話說：** 就像紙娃娃——一個平面的角色圖，可以換衣服換動作。2D 遊戲裡你看到的每一個角色、每一個道具、每一顆子彈，都是一張張的 Sprite。

---

## 它到底是什麼？

Sprite 是 2D 遊戲中用來顯示角色、物件、特效的「圖片」。你在超級瑪利歐裡看到的瑪利歐、蘑菇、金幣、烏龜⋯⋯每一個都是一張 Sprite。

一個角色通常不只有一張 Sprite，而是一整套。比如「站著」是一張、「走路」是好幾張（不同步伐的樣子）、「跳躍」是一張、「死掉」又是一張。遊戲在運行時，根據角色的狀態快速切換不同的 Sprite，就會產生「動畫」的效果——就像翻書動畫。

這些 Sprite 通常會被打包成一張大圖，叫做 Sprite Sheet（精靈圖表）。把所有動作圖排列在同一張大圖上，遊戲引擎需要哪個就裁哪個區塊來顯示，這樣比載入一百張小圖有效率多了。

## 生活比喻 / 實際例子

還記得紙娃娃嗎？一個紙做的人型，配上好幾套紙做的衣服，你把不同的衣服「蓋」在人型上，它就「換裝」了。Sprite 的概念很像——遊戲引擎根據不同情況，顯示不同的圖片，讓角色看起來在「動」。

或者想想老式的動畫片：每一格膠卷就是一張圖，快速播放就變成動畫。2D 遊戲的角色動畫就是快速切換不同的 Sprite 圖片。

**造句**：「美術那邊的走路 **Sprite** 還沒出完，角色目前只有站立和跳躍的動畫。」

## 為什麼要知道這個詞？

- 所有 2D 遊戲都在用 Sprite，從超級瑪利歐到手機遊戲全部都是
- 遊戲企劃在規劃角色時，需要列出「這個角色需要哪些 Sprite」（站立、走路、攻擊、受傷⋯⋯）
- 跟 2D 美術溝通時，「Sprite 尺寸」「Sprite Sheet」「Sprite 動畫張數」是常見的溝通內容
- 理解 Sprite 才能理解 [Pixel Art](pixel-art.md) 和 [Tilemap](tilemap.md) 的概念
