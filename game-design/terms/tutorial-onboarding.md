# Tutorial / Onboarding（新手教學）— 遊戲一開始教你怎麼玩的引導流程

> **白話說：** 就像第一天上班有人帶你認識環境、教你打卡，讓你不會一臉茫然。

---

## 它到底是什麼？

Tutorial 就是遊戲一開始的「新手教學時間」。當你打開一款新遊戲，畫面上會跳出提示告訴你「按 A 跳躍」「按 B 攻擊」「往這邊走」，這些引導你學會基本操作的流程，就是 Tutorial。Onboarding 則是更廣義的說法，涵蓋整個「讓新玩家上手」的過程。

好的新手教學會讓你在玩的過程中自然學會所有操作，不會覺得在上課。壞的新手教學就像那種塞給你一本 50 頁操作手冊要你先讀完才能開始——你還沒玩就先累了。

現在的遊戲越來越流行「邊玩邊教」的設計。與其在開頭放一堆文字說明，不如設計一個簡單的關卡，讓你在過關的過程中自然而然學會所有技能。就像學騎腳踏車，不是先聽一小時的物理課，而是直接跨上去騎，旁邊有人扶著你。

## 生活比喻 / 實際例子

想想你第一天去新公司上班。好的公司會安排一個同事帶你：「這是你的座位、這是茶水間、中午大家都去那邊吃飯、打卡機在門口、有問題找這個人。」你一天下來就大概知道怎麼運作了。這就是好的 Onboarding。

壞的 Onboarding 呢？就是第一天上班沒人理你，桌上放一疊文件叫你自己看，你連廁所在哪都不知道。很多遊戲的新手教學也是這樣——開頭丟一堆操作說明視窗，玩家根本記不住。

你有沒有用過 LINE 或 Instagram？第一次打開 App 的時候，它會一步步引導你設定大頭貼、加好友、發第一則貼文。這整個流程就是一種 Onboarding 設計。

**造句**：「這遊戲的 Tutorial 做得很棒，我完全沒看說明書就學會怎麼玩了。」

## 為什麼要知道這個詞？

- 你玩新遊戲時覺得「好上手」或「搞不懂怎麼玩」，差別通常就在 Tutorial 設計
- 手機遊戲特別重視 Onboarding，因為玩家沒耐心——教學太難，直接刪 App
- 這個概念不只用在遊戲，App、網站、甚至實體服務都在做 Onboarding 設計
- 看遊戲評論時常會看到「新手引導友善」或「上手門檻太高」這類評價
