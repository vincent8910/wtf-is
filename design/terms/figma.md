# Figma — 設計師最常用的線上協作設計工具

> **白話說：** Figma 就像設計界的 Google Docs——大家可以同時打開同一個設計檔，設計師在畫圖、PM 在旁邊留言、工程師在看尺寸，全部即時同步。

---

## 它到底是什麼？

**Figma** 是一款**線上設計工具**，用瀏覽器就能打開，不用安裝軟體。它是目前全世界設計師最主流的工具。

Figma 能做什麼：
- 畫 [Wireframe](wireframe.md)（線框圖）
- 做 [Mockup](mockup.md)（視覺稿）
- 做 [Prototype](prototype.md)（互動原型）
- 建立 [設計系統（Design System）](design-system.md)
- [切圖](slicing.md)（工程師可以直接匯出素材）
- 即時多人協作（像 Google Docs 一樣）

為什麼 Figma 這麼受歡迎：
| 優點 | 說明 |
|------|------|
| 即時協作 | 多人同時編輯，不用傳檔案來傳去 |
| 瀏覽器就能用 | 不用安裝，打開連結就能看 |
| 非設計師也能參與 | PM、工程師可以直接在設計稿上留言、查看標注 |
| 免費方案夠用 | 個人使用完全免費 |

## 生活比喻 / 實際例子

以前的設計流程像傳統寫信：
- 設計師在自己電腦上畫好 → 存成檔案 → email 寄出去 → 等對方下載打開 → 回覆修改意見 → 改完再寄一次⋯⋯

用 Figma 就像開視訊會議：
- 設計師分享一個連結 → 大家同時看到同一個畫面 → 直接在上面留言 → 設計師當場修改 → 大家即時看到更新

實際場景：
- 設計師說：「**Figma** 連結丟在 Slack 上了，你們直接看。」
- PM 說：「我在 **Figma** 上留了幾個 comment，你看一下。」
- 工程師說：「我從 **Figma** 上直接量間距和匯出圖片就好。」
- 老闆說：「打開 **Figma** 讓我看一下現在做到哪了。」

## 為什麼要知道這個詞？

- Figma 是目前設計協作的標準工具，你的團隊十之八九在用
- 就算你不是設計師，也需要在 Figma 上看設計稿、留言、確認
- 學會在 Figma 上留 comment 和查看標注，比 email 來回修改有效率 100 倍
- 它完全免費就能使用（只要有連結就能看），門檻非常低
