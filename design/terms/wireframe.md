# Wireframe — 只有框框和線條的草圖

> **白話說：** Wireframe 就像蓋房子前的平面圖——只畫出「哪裡放客廳、哪裡放廚房」，還沒選磁磚、還沒挑窗簾。重點是位置和功能，不是好不好看。

---

## 它到底是什麼？

**Wireframe**（線框圖）是設計的第一步，它只用簡單的框框、線條和文字來表示頁面的結構。

Wireframe 裡面：
- **有**：方框代表圖片位置、線條代表文字、按鈕的位置和大小
- **沒有**：顏色、真正的圖片、漂亮的字體、細節裝飾

```
┌────────────────────────────┐
│          [Logo]    [選單]    │
├────────────────────────────┤
│                            │
│       [ 大圖 Banner ]       │
│                            │
├──────────┬─────────────────┤
│  [產品1]  │  [產品2]  [產品3] │
│  xxxxxx  │  xxxxxx  xxxxxx │
│  [按鈕]   │  [按鈕]   [按鈕]  │
└──────────┴─────────────────┘
```

這就是一個簡單的 Wireframe。你可以很快看出「首頁上面是 Logo 和選單，中間是大圖，下面是三個產品」。

## 生活比喻 / 實際例子

想像你要辦一場婚宴：

- **Wireframe** = 場地配置圖：「這邊放主桌、那邊放舞台、中間是走道」
- **[Mockup](mockup.md)** = 3D 效果圖：看到花的顏色、桌巾的質感、燈光效果
- **[Prototype](prototype.md)** = 試走一遍：新郎新娘真的走一次看動線順不順

Wireframe 的重點不是漂亮，而是「東西放對位置了嗎」。

實際場景：
- 設計師說：「我先出個 **Wireframe**，你們確認架構沒問題再往下做。」
- PM 說：「**Wireframe** 看起來沒問題，可以開始做 Mockup 了。」
- 老闆看到 Wireframe 說：「怎麼這麼醜？」——因為它本來就不是拿來看漂亮的。

## 為什麼要知道這個詞？

- Wireframe 是整個設計流程的起點，看到它不要嫌醜
- 在 Wireframe 階段修改是最便宜的——移一個框框只要幾秒鐘，但做成 [Mockup](mockup.md) 再改就要花好幾小時
- 這是跟設計師溝通「頁面要放什麼東西、放在哪裡」最有效率的工具
- 了解 Wireframe → [Mockup](mockup.md) → [Prototype](prototype.md) 的流程，才知道每個階段該給什麼回饋
