# 切圖 — 把設計稿切成一塊一塊交給工程師

> **白話說：** 切圖就像把一張大海報裁成一小塊一小塊——設計師把畫好的設計稿，把裡面的圖片、icon、按鈕一個一個「切」出來，交給工程師去組裝成真正的網頁或 App。

---

## 它到底是什麼？

**切圖**（Slicing）是設計師把完成的設計稿，將裡面的視覺元素一個一個匯出成獨立的圖片檔案，讓工程師可以拿去用。

設計師的設計稿是一張「完整的畫」，但工程師需要的是「一個一個零件」：

```
設計稿（一張完整的畫面）
├── Logo.png      ← 切出來的 Logo
├── banner.jpg    ← 切出來的首頁大圖
├── icon-cart.svg  ← 切出來的購物車圖示
├── btn-buy.png    ← 切出來的購買按鈕
└── bg-pattern.png ← 切出來的背景圖案
```

工程師拿到這些「零件」，再用程式碼把它們排列組合成真正的網頁或 App。

## 生活比喻 / 實際例子

想像你要做一本剪貼簿：

- 你有一本**時尚雜誌**（= 設計稿）
- 你把裡面的漂亮圖片一張張**剪下來**（= 切圖）
- 然後再把剪下來的圖片**貼到你的剪貼簿上排好**（= 工程師組裝成網頁）

實際場景：
- 設計師說：「我 [設計稿](comp.md) 畫好了，今天會把**切圖**給工程師。」
- 工程師說：「這個 icon 的**切圖**可以給我 SVG 格式嗎？這樣放大不會糊。」
- PM 催進度：「**切圖**什麼時候可以給？工程那邊在等。」

## 為什麼要知道這個詞？

- 切圖是設計交給工程的關鍵交接點，時程上要預留時間
- 切圖品質會影響最終成品——切得不好，網頁或 App 看起來就不精緻
- 現在用 [Figma](figma.md) 等工具，工程師可以自己直接匯出素材，切圖流程比以前輕鬆很多
- 知道這個詞，你就能理解為什麼設計師畫完稿之後還需要時間才能「交件」
