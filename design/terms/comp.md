# 設計稿（Comp）— 設計師交給工程師的最終版設計圖

> **白話說：** 設計稿就像建築師交給施工隊的藍圖——上面標好了每一個尺寸、顏色、間距，工程師照著做就對了。

---

## 它到底是什麼？

**Comp**（Comprehensive Layout 的縮寫，又叫「完稿」或「設計稿」）是設計流程的最終產出。它包含了所有開發需要的細節：

- 每個元素的精確尺寸和位置
- 正確的顏色色碼（例如 #FF5733）
- 字體名稱、[字級、字距、行距](typography.md)
- 各種狀態（按鈕的預設、懸停、按下、禁用的樣子）
- 手機版、平板版、電腦版的不同[排版](layout.md)

設計稿是從 [Wireframe](wireframe.md) → [Mockup](mockup.md) → [Prototype](prototype.md) 一路修改確認下來的最終版本。

## 生活比喻 / 實際例子

想像你要訂做一件西裝：

- **Wireframe** = 跟裁縫說「我要一件三件式的」（大方向）
- **Mockup** = 看布料色卡、選扣子款式（視覺決定）
- **Prototype** = 試穿半成品，看合不合身（體驗測試）
- **Comp（設計稿）** = 最終確認的版型圖，標好每一個尺寸——裁縫照著做就對了

設計稿就是那張「照著做就對了」的圖。

實際場景：
- 設計師說：「**設計稿**交出去了，[切圖](slicing.md) 也附在 [Figma](figma.md) 上了。」
- PM 說：「等 **Comp** 確認後才能開工程的票。」
- 工程師說：「**設計稿**上這個間距是 16px 還是 24px？標示不清楚。」

## 為什麼要知道這個詞？

- 設計稿是設計團隊和工程團隊之間最重要的交接文件
- 設計稿的品質直接決定工程做出來的東西跟設計的一不一致
- 如果設計稿標示不清楚，工程師就會自己猜，做出來跟設計師想的不一樣
- 了解整個設計流程（Wireframe → Mockup → Prototype → Comp），你就知道催設計師的時候該催到哪個階段
