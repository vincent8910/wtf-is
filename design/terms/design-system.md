# 設計系統（Design System）— 整理好的一套設計規則和元件庫

> **白話說：** 設計系統就像麥當勞的品牌手冊——從 Logo 怎麼用、菜單排版、店員制服到杯子上的字體，全部都有統一規定。這樣不管開到哪裡，每家店看起來都是「麥當勞」。

---

## 它到底是什麼？

**設計系統（Design System）** 是一套完整的設計規範 + 現成的元件庫，包含：

| 包含什麼 | 舉例 |
|---------|------|
| **顏色規範** | 品牌色 #FF5733、輔助色、背景色 |
| **字體規範** | 標題用 Noto Sans Bold 24px、內文用 16px |
| **元件庫** | 按鈕、輸入框、卡片、彈窗⋯⋯都做好了 |
| **間距規則** | 元素之間間距用 8 的倍數（8px、16px、24px） |
| **圖示風格** | 統一的線條粗細、圓角大小 |
| **使用規範** | 什麼時候用什麼顏色、什麼場景用什麼元件 |

有了設計系統：
- **設計師**不用每次從零開始，直接拿現成元件來排
- **工程師**不用每次重新刻，直接套用寫好的元件
- **整個產品**不管是哪個頁面、哪個功能，風格都一致

## 生活比喻 / 實際例子

想像你開了一家連鎖手搖飲店：

- **沒有設計系統**：每家分店自己決定招牌怎麼做、杯子什麼顏色、菜單怎麼排——客人走進去根本不知道是同一家店
- **有設計系統**：所有分店的招牌、杯子、菜單、制服都照手冊來——不管去哪家店，一看就知道是同一個品牌

知名的設計系統：
- Google 的 **Material Design**
- Apple 的 **Human Interface Guidelines**
- IBM 的 **Carbon Design System**

實際場景：
- 設計師說：「我們已經有 **Design System** 了，新頁面直接套元件就好。」
- PM 說：「這個按鈕不在 **Design System** 裡，要先跟設計確認要不要新增。」
- 工程師說：「**Design System** 裡有這個元件，我直接用就好。」

## 為什麼要知道這個詞？

- 設計系統是大型產品維持一致性的關鍵
- 有設計系統，新功能的設計和開發速度都會快很多
- 當你發現「A 頁面的按鈕長這樣、B 頁面的按鈕長那樣」，就表示需要一個設計系統
- 跟 [CI / VI](ci-vi.md) 有關聯——CI/VI 是品牌層面的規範，設計系統是產品層面的規範
