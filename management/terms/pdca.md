# PDCA — 計畫→執行→檢查→改進，不斷循環的做事方法

> **白話說：** PDCA 就像**學騎腳踏車**——先想怎麼騎（Plan）、實際騎看看（Do）、摔了想想哪裡不對（Check）、調整姿勢再試一次（Act）。一直重複到學會為止。

---

## 它到底是什麼？

**PDCA** 是四個英文字的縮寫：

| 階段 | 英文 | 中文 | 白話說 |
|------|------|------|--------|
| **P** | Plan | 計畫 | 先想好要做什麼、怎麼做 |
| **D** | Do | 執行 | 照計畫去做 |
| **C** | Check | 檢查 | 做完看看結果怎樣 |
| **A** | Act | 改進 | 哪裡不好就改，然後重新來一輪 |

重點在「循環」這兩個字——不是做一次就結束，而是**做完一輪再來一輪**，每次都比上次更好。

這方法最早是品質管理大師戴明（Deming）推廣的，所以也叫「戴明循環」。

## 生活比喻 / 實際例子

假設你開了一間手搖飲料店，想提高來客數：

```
第一輪 PDCA：
├─ Plan：決定推出「買一送一」活動
├─ Do：這週開始執行買一送一
├─ Check：發現來客數增加了，但利潤反而下降
└─ Act：改成「第二杯半價」，利潤跟來客數都能顧到

第二輪 PDCA：
├─ Plan：用「第二杯半價」再試一週
├─ Do：執行新方案
├─ Check：來客數跟利潤都有提升！
└─ Act：把這個方案變成固定活動
```

日常生活也是一樣：

- **煮飯**：第一次太鹹 → 下次少放一點鹽 → 剛好 → 記住這個比例
- **考試準備**：讀完考差了 → 檢討哪個章節沒讀好 → 下次加強那部分

在公司裡你會聽到：

- 「我們用 **PDCA** 的方式來跑這個專案」→ 邊做邊改善，不追求一次到位
- 「這次的 **Check** 結果怎麼樣？」→ 檢查完結果如何？
- 「進到 **Act** 階段了」→ 根據結果來調整做法

## 為什麼要知道這個詞？

PDCA 是最基本的管理思維之一，很多公司的工作流程都是建立在這個概念上：

- [SOP](sop.md) 就是 PDCA 跑了很多輪之後，把最佳做法固定下來的結果
- [OKR](okr.md) 每季檢視進度並調整，本質上也是 PDCA 的精神
- 很多會議裡的「回顧」（Retrospective）就是 PDCA 的 Check + Act

知道 PDCA，你就理解為什麼公司總是在「改善、優化、迭代」——因為做事情本來就不是一次就完美，重點是每次都變好一點。
