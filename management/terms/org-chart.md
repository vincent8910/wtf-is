# 組織架構（Org Chart） — 誰管誰、誰跟誰同部門的那張圖

> **白話說：** Org Chart 就像**家族族譜**——誰是誰的上級、誰跟誰是同輩、哪些人屬於同一個分支，畫成一張圖一看就懂。

---

## 它到底是什麼？

**Org Chart** 是 **Organization Chart** 的縮寫，中文叫「組織架構圖」。

就是一張圖，畫出公司裡：

- **誰管誰**（匯報關係）
- **有哪些部門**（組織單位）
- **每個人在哪個位置**（職位層級）

一個典型的 Org Chart 長這樣：

```
                    執行長（CEO）
                        │
          ┌─────────────┼─────────────┐
          │             │             │
    技術長（CTO）   營運長（COO）  財務長（CFO）
          │             │             │
    ┌─────┤       ┌─────┤         會計部
    │     │       │     │
  前端組  後端組  業務部  客服部
```

常見的組織架構類型：

| 類型 | 特色 | 適合 |
|------|------|------|
| 功能型 | 按專業分（工程部、行銷部） | 中小型公司 |
| 事業部型 | 按 [BU](bu.md) 分（手機事業部、雲端事業部） | 大型多角化公司 |
| 矩陣型 | 同時有部門主管和專案主管 | 需要大量[跨部門協作](cross-functional.md)的公司 |

## 生活比喻 / 實際例子

就像一所學校的組織：

```
校長
├─ 教務處
│  ├─ 一年級導師群
│  ├─ 二年級導師群
│  └─ 三年級導師群
├─ 學務處
│  ├─ 生活輔導組
│  └─ 體育組
└─ 總務處
   ├─ 出納
   └─ 庶務
```

你看了就知道：導師歸教務處管、體育老師歸學務處管、要修水電找總務處。

在公司裡你會聽到：

- 「看一下 **Org Chart** 就知道找誰」→ 看組織架構圖就知道該聯繫誰
- 「公司**組織架構**要調整了」→ 部門要合併、拆分或重組
- 「你的 **reporting line** 是誰？」→ 你向誰匯報？（你的直屬主管是誰）

## 為什麼要知道這個詞？

Org Chart 是你在公司生存的「地圖」：

- **新人到職**：第一件事就是看 Org Chart，搞清楚公司有哪些部門、你在哪個位置
- **找人辦事**：Org Chart 告訴你該找哪個部門的誰
- **理解決策**：知道誰是誰的主管，就能理解決策是怎麼做出來的
- **職涯規劃**：Org Chart 讓你看到可能的升遷路線和橫向發展機會

當公司說「組織重整」或「改組」，就是在重畫這張圖——而這通常意味著有人要換位置、換主管、甚至離開。
