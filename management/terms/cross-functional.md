# 跨部門協作（Cross-functional） — 不同部門的人一起合作完成一件事

> **白話說：** 跨部門協作就像**拍一部電影**——導演、編劇、演員、攝影師、燈光師、配樂師⋯⋯每個人的專業不同，但要一起合作才能把電影拍出來。

---

## 它到底是什麼？

**Cross-functional** 直翻是「跨功能」，在公司裡就是指**不同部門的人組成一個臨時團隊，一起做一件事**。

為什麼需要跨部門？因為很多事情不是一個部門就能搞定的。

舉例：公司要推出一個新產品

```
跨部門團隊：
├─ 產品部：決定要做什麼功能
├─ 工程部：把功能開發出來
├─ 設計部：設計使用介面
├─ 行銷部：規劃上市宣傳
├─ 業務部：準備銷售話術
├─ 客服部：準備回答客戶問題
└─ 法務部：確認合規和隱私條款
```

每個部門各自有專業，但這件事需要大家一起配合才做得成。

## 生活比喻 / 實際例子

就像辦一場公司尾牙：

| 誰 | 負責什麼 |
|----|---------|
| 行政 | 訂場地、安排座位 |
| HR | 設計抽獎、員工活動 |
| 行銷 | 做主視覺、拍照錄影 |
| IT | 架投影設備、音響 |
| 財務 | 控制預算 |
| 老闆 | 上台致詞、發紅包 |

沒有任何一個部門可以獨自辦完尾牙，大家一起才行——這就是跨部門協作。

在公司裡你會聽到：

- 「這是一個 **cross-functional** 的專案」→ 需要好幾個部門一起做
- 「組一個**跨部門**小組來處理」→ 從各部門抽人出來組成臨時團隊
- 「**跨部門**溝通好累」→ 每個部門的語言和優先順序不一樣，溝通成本很高

## 為什麼要知道這個詞？

跨部門協作是現代公司裡最常見（也最容易卡關）的工作方式：

- **工程師**：你常常需要跟 PM、設計、QA 協作，這就是跨部門
- **PM**：你的日常工作幾乎就是在做跨部門協調
- **任何人**：公司越大，跨部門的需求越多

跨部門的挑戰：

- 每個部門的 [KPI](kpi.md) 不同，優先順序可能衝突
- 溝通語言不同（工程師講技術、業務講數字、設計講體驗）
- 沒有人直接管所有人，很靠協調和溝通能力

搞定跨部門協作，需要清楚的 [Stakeholder](stakeholder.md) 管理和明確的分工。
