# 利害關係人（Stakeholder） — 跟這件事有關的所有人

> **白話說：** Stakeholder 就像**辦一場婚禮時要顧到的所有人**——新人、雙方父母、伴郎伴娘、賓客、婚禮場地、攝影師、主持人⋯⋯每個人都有不同的期待和需求，你都不能忽略。

---

## 它到底是什麼？

**Stakeholder** 翻成中文叫「利害關係人」，白話說就是：**跟這件事有利害關係的所有人。**

不只是你的老闆或客戶，而是所有**會被這件事影響、或能影響這件事**的人。

舉例：公司要上線一個新功能，Stakeholder 可能有：

```
這個新功能的 Stakeholder：
├─ 👤 PM（決定功能規格）
├─ 👥 工程團隊（實際開發）
├─ 🎨 設計師（設計介面）
├─ 📣 行銷（要規劃上線宣傳）
├─ 📞 客服（要準備回答用戶問題）
├─ 💰 老闆（要批准預算）
├─ 👩‍💼 業務（可能拿來跟客戶談）
└─ 👥 用戶（最終使用這個功能的人）
```

## 生活比喻 / 實際例子

想像你決定要搬家：

| Stakeholder | 他為什麼有關 |
|-------------|-------------|
| 你自己 | 要住得舒服 |
| 房東 | 你要跟他解約 |
| 新房東 | 你要跟他簽約 |
| 室友 | 會影響他的生活 |
| 公司 | 通勤距離可能改變 |
| 家人 | 可能要幫忙搬家 |
| 鄰居 | 搬家那天可能會吵到他們 |

你不用每個人都聽他的意見，但你需要**知道有這些人存在**，才不會漏掉什麼重要的事。

在公司裡你會聽到：

- 「這個專案的 **Stakeholder** 有哪些人？」→ 誰跟這件事有關？
- 「要先跟 **Stakeholder** 對齊」→ 先讓所有相關的人都知道狀況並達成共識
- 「**Stakeholder** 太多了，很難推動」→ 相關的人太多，每個人意見都不同

## 為什麼要知道這個詞？

Stakeholder 是職場上非常重要的概念：

- **開始做任何專案之前**：先搞清楚 Stakeholder 有誰，才不會做了半天發現漏掉重要的人
- **溝通的時候**：不同 Stakeholder 在意的事不同，跟老闆講成本效益、跟工程師講技術可行性
- **[跨部門協作](cross-functional.md)**：越大的專案，Stakeholder 越多，溝通越複雜
- **做決策的時候**：要考慮各方 Stakeholder 的利益，不能只顧一方

一句話總結：**事情做不做得成，往往不只看技術，更看你有沒有搞定所有 Stakeholder。**
