# Kanban（看板）— 把任務分成「待做、進行中、完成」的視覺化管理板

> **白話說：** 就像餐廳廚房的訂單夾——新訂單夾在左邊、正在做的在中間、做好等出餐的在右邊。一眼就知道所有事情的狀態。

---

## 它到底是什麼？

Kanban 是一種**視覺化的工作管理方法**。它的核心概念超簡單：把工作分成幾個欄位，每張卡片代表一個任務，隨著進度從左邊移到右邊。

最基本的 Kanban 只有三欄：

```
| 待做 (To Do) | 進行中 (In Progress) | 完成 (Done) |
|--------------|---------------------|-------------|
| 設計首頁      | 寫登入功能            | 建立資料庫    |
| 修改 LOGO    | 測試購物車            |             |
| 串接金流      |                     |             |
```

Kanban 最大的好處是**一眼就能看出所有工作的狀態**。哪些事還沒開始、哪些正在做、哪些已經完成，全部一目了然。

另一個重點是 **WIP 限制**（Work In Progress Limit）——限制「進行中」的數量。比如規定同時只能有 3 件事在進行中，避免大家同時做太多事反而什麼都做不好。

常見的 Kanban 工具有 Trello、Jira、Notion，或者最原始的方式——在白板上貼便利貼。

## 生活比喻 / 實際例子

你在打掃房子，在客廳白板上貼了三排便利貼：

- **還沒做**：洗衣服、拖地、擦窗戶、整理書櫃
- **正在做**：洗碗（你在洗）、吸地板（另一半在吸）
- **做完了**：倒垃圾 ✅、擦桌子 ✅

做完一件事就把便利貼從「正在做」移到「做完了」，再從「還沒做」拿一張新的來做。

**造句**：「把這個 bug 加到 **Kanban** 的『待做』，我現在手上還有兩個在進行中。」

## 為什麼要知道這個詞？

- 很多團隊用 Kanban 來管理日常工作，你會每天看到它
- Trello 就是一個典型的 Kanban 工具
- 跟 [Sprint](sprint.md) 不同，Kanban 不強制固定時間週期，任務流是持續的
- 有些團隊會混合使用 Scrum + Kanban，叫做 Scrumban
