# Gantt Chart（甘特圖）— 用長條圖顯示每個任務的時間和進度

> **白話說：** 就像學校的課表——一目了然看到什麼時候做什麼事、做多久、跟誰的時間重疊。

---

## 它到底是什麼？

Gantt Chart（甘特圖）是一種**視覺化的專案時間表**。它用橫向的長條圖來顯示每個任務「什麼時候開始、什麼時候結束」，以及任務之間的前後關係。

看一張甘特圖，你馬上能知道：
- 現在有哪些任務在進行中
- 哪些任務已經延遲了
- 哪些任務要等別人做完才能開始
- 整個專案預計什麼時候完成

甘特圖大概長這樣（簡化版）：

```
任務 A：████████
任務 B：    ████████████
任務 C：            ████████
任務 D：                ████████████
        |----3月----|----4月----|----5月----|
```

你可以清楚看到任務 B 要等任務 A 做到一半才開始，任務 C 要等任務 B 做完才能開工。

常見的甘特圖工具有 Jira、Asana、Monday.com，或最簡單的就是用 Excel 畫。

## 生活比喻 / 實際例子

你在規劃一場婚禮，列出所有要做的事和時間：

```
訂場地：  ██████（1月）
設計喜帖：    ████（1月底～2月初）
寄喜帖：      ████（2月）
訂餐廳：  ████████（1月～2月）
婚紗拍攝：        ████████（2月～3月）
彩排：                    ██（3月底）
婚禮：                      ◆（4月1日）
```

這就是一張甘特圖！你一眼就能看出什麼事該先做、什麼可以同時進行、什麼不能延遲。

**造句**：「你可以拉一下 **甘特圖** 給客戶看，讓他知道每個階段的時間和進度。」

## 為什麼要知道這個詞？

- PM 最常用的視覺化工具之一，跟客戶或老闆報告進度時很好用
- 能快速看出「時間衝突」和「延遲風險」
- 搭配 [Milestone](milestone.md) 使用，重要節點一目了然
- 比起純文字的進度報告，甘特圖讓不懂技術的人也看得懂
