## 這是什麼？

這是一份給**沒有專案管理背景的人**看的術語翻譯手冊。

如果你是工程師、設計師、行銷、老闆，或任何每天在跑 Sprint 卻搞不清楚這些詞到底什麼意思的人——這裡就是你的救星。

每個術語都有：
- **白話解釋**：不用任何專業背景就能懂
- **生活比喻**：用你熟悉的事物來對照
- **造句示範**：讓你知道這個詞在什麼情境下會出現

## 敏捷開發類

> 跑 Scrum 的團隊每天都會遇到的詞，搞懂這些就能跟上節奏。

| 詞彙 | 一句話解釋 | 比喻 | 你會在哪聽到 | 詳細說明 |
|------|-----------|------|-------------|---------|
| Sprint（衝刺） | 把大工作切成一小段一小段來做，通常兩週一輪 | 馬拉松切成短跑衝刺 | 「這個 **Sprint** 的目標是做完登入功能。」 | [→ 詳細](terms/sprint.md) |
| Backlog（待辦清單） | 所有想做但還沒做的事情的清單 | 冰箱上貼的「待買清單」 | 「先丟進 **Backlog**，下個 Sprint 再排。」 | [→ 詳細](terms/backlog.md) |
| User Story（使用者故事） | 用使用者的角度描述一個功能需求 | 點餐說「不太甜的拿鐵」 | 「這個功能的 **User Story** 是什麼？」 | [→ 詳細](terms/user-story.md) |
| Stand-up（站立會議） | 每天花 15 分鐘站著快速報告進度 | 早餐店老闆交代今天備料 | 「每天早上 10 點有 **Stand-up**。」 | [→ 詳細](terms/stand-up.md) |
| Retrospective（回顧會議） | 做完一輪後回顧什麼做得好、什麼要改 | 球賽後看比賽影片檢討 | 「禮拜五下午開 **Retro**。」 | [→ 詳細](terms/retrospective.md) |
| Velocity（速度 / 產能） | 團隊每個 Sprint 能完成多少工作量 | 工廠的日產量 | 「我們 **Velocity** 大概每個 Sprint 25 點。」 | [→ 詳細](terms/velocity.md) |

## 規劃與追蹤類

> 怎麼把大目標拆小？怎麼追蹤進度？PM 吃飯的工具都在這。

| 詞彙 | 一句話解釋 | 比喻 | 你會在哪聽到 | 詳細說明 |
|------|-----------|------|-------------|---------|
| Epic（史詩） | 一個很大的功能需求，要拆成好幾個小任務 | 蓋一棟房子 | 「『購物車功能』是一個 **Epic**。」 | [→ 詳細](terms/epic.md) |
| Milestone（里程碑） | 專案中的重要檢查點，代表完成了一個階段 | 爬山途中的休息站 | 「下週五是 **Milestone**，設計稿要定稿。」 | [→ 詳細](terms/milestone.md) |
| Gantt Chart（甘特圖） | 用長條圖顯示每個任務的時間和進度 | 學校的課表 | 「拉一下**甘特圖**給客戶看進度。」 | [→ 詳細](terms/gantt-chart.md) |
| Kanban（看板） | 把任務分成「待做、進行中、完成」的視覺化管理板 | 餐廳廚房的訂單夾 | 「把這個 bug 加到 **Kanban** 的待做欄。」 | [→ 詳細](terms/kanban.md) |

## 風險與方法論

> 專案為什麼會 delay？不同的開發方法有什麼差別？

| 詞彙 | 一句話解釋 | 比喻 | 你會在哪聽到 | 詳細說明 |
|------|-----------|------|-------------|---------|
| Scope Creep（範圍蔓延） | 專案做著做著，需求越加越多，怎麼做都做不完 | 裝潢客廳變成整棟翻新 | 「這不在 Scope 裡，再加就是 **Scope Creep**。」 | [→ 詳細](terms/scope-creep.md) |
| Waterfall（瀑布式開發） | 一階一階往下做，做完一階才能做下一階 | 蓋房子：地基→牆壁→裝潢 | 「這個標案用 **Waterfall** 做。」 | [→ 詳細](terms/waterfall.md) |

## 比喻對照總表

> 每個專案管理術語，用一句生活比喻讓你秒懂。

### 敏捷開發類

| 術語 | 比喻 |
|------|------|
| Sprint（衝刺） | **馬拉松切成短跑衝刺**——每跑完一段就休息、看方向 |
| Backlog（待辦清單） | **冰箱上貼的待買清單**——想到就加，但要看優先順序 |
| User Story（使用者故事） | **點餐說「不太甜的拿鐵」**——用需求方的語言，不用做法的語言 |
| Stand-up（站立會議） | **早餐店老闆交代今天備料**——兩分鐘搞定，站著不囉唆 |
| Retrospective（回顧會議） | **球賽後看比賽影片檢討**——不是怪罪誰，而是一起變更好 |
| Velocity（速度 / 產能） | **工廠的日產量**——知道一天能做幾台車，才知道訂單要排多久 |

### 規劃與追蹤類

| 術語 | 比喻 |
|------|------|
| Epic（史詩） | **蓋一棟房子**——太大了，要拆成挖地基、蓋牆壁、裝水電 |
| Milestone（里程碑） | **爬山途中的休息站**——還沒到山頂，但知道自己在對的路上 |
| Gantt Chart（甘特圖） | **學校的課表**——什麼時候做什麼事、做多久、一目了然 |
| Kanban（看板） | **餐廳廚房的訂單夾**——新單、做菜中、出餐，一眼看完 |

### 風險與方法論

| 術語 | 比喻 |
|------|------|
| Scope Creep（範圍蔓延） | **裝潢客廳變成整棟翻新**——每個「順便」加起來就是災難 |
| Waterfall（瀑布式開發） | **蓋房子：地基→牆壁→裝潢**——一步一步來，不能跳著做 |
