# Rate Limit（頻率限制）— 每人限購兩杯

> **白話說：** Rate Limit 就像飲料店貼公告「每人每天限購 2 杯」——不是不賣你，只是怕你一個人把庫存買光，其他客人都喝不到。

---

## 它到底是什麼？

**Rate Limit（頻率限制）** 是伺服器用來保護自己的一種機制：**限制你在一段時間內能發多少次請求。**

為什麼要限制？因為伺服器的能力有限。如果有人瘋狂發送請求（不管是惡意攻擊還是程式寫錯），伺服器會被搞到當掉，所有人都用不了。

```
沒有 Rate Limit：
  某個用戶一秒打 10,000 次 API → 伺服器炸掉 → 全部用戶都受影響

有 Rate Limit：
  每個用戶一分鐘最多 60 次 → 超過的直接拒絕 → 伺服器活得好好的
```

就像好市多的衛生紙特價——不限購的話，第一個人搬 200 包走，後面的人只能看空貨架。限購之後，大家都買得到。

## 生活比喻 / 實際例子

Rate Limit 在你日常生活中到處都有：

| 場景 | Rate Limit |
|-----|-----------|
| 7-11 集點活動 | 每帳號每天限兌換 1 次 |
| 搶演唱會門票 | 每 IP 每分鐘最多嘗試 5 次 |
| 免費 AI 工具 | 每天限使用 20 次 |
| 手機輸入密碼 | 錯 5 次就鎖定 |

你用 Claude、ChatGPT 或其他 AI 工具時，如果看到「你今天的使用量已達上限」，那就是被 Rate Limit 限制住了。

**造句：**「我們的 [API](api.md) 有 **Rate Limit**，每分鐘最多 100 次請求。如果你的程式一直收到 429 錯誤，就是超過限制了。」

## 為什麼要知道這個詞？

- 使用任何 [API](api.md) 服務時，一定會碰到 Rate Limit 的規定
- 如果你的程式突然出現 `429 Too Many Requests` 錯誤，就是踩到 Rate Limit 了
- PM 在規劃功能時需要考慮：「如果這個功能每秒會呼叫第三方 API 100 次，人家的 Rate Limit 夠嗎？」
- Rate Limit 也是一種安全防護——防止有人用自動程式瘋狂嘗試密碼（暴力破解）
