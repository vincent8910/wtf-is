# Landing Page（到達頁） — 使用者點了廣告或連結後「降落」的第一個頁面

> **白話說：** 就像百貨公司為了週年慶特別佈置的一樓大廳——目標就是讓你一進來就想買東西。

---

## 它到底是什麼？

Landing Page 就是使用者點了某個連結（廣告、Email、社群貼文）後看到的第一個頁面。

它跟一般網頁不同的地方在於：它只有一個目標，通常是要你做一件特定的事：
- 填表單留下聯絡資訊
- 按下「立即購買」
- 註冊帳號
- 下載 App

所以 Landing Page 通常長得很簡潔：一個大標題、一段吸引人的文案、一個明顯的按鈕。不會像首頁一樣放一大堆選單和連結，因為怕你分心。

## 生活比喻 / 實際例子

想像你發傳單宣傳新餐廳：

- **一般首頁** = 客人走進一個大商場，到處逛，可能逛一逛就走了
- **Landing Page** = 客人拿到傳單，上面寫「憑券到櫃檯免費試吃」，直接帶你到試吃攤位

Landing Page 的設計就是要「減少選擇 → 引導行動」。

實際例子：
- 行銷說：「這波廣告需要一個 **landing page**，專門介紹新方案」
- PM 說：「**Landing page** 的[轉換率](conversion-rate.md)只有 2%，要優化一下」
- 設計師問：「首頁還是 **landing page**？」行銷：「是 landing page，只要放報名表單就好」

## 為什麼要知道這個詞？

- 行銷投廣告時，幾乎都需要搭配一個專屬的 Landing Page
- Landing Page 的好壞直接影響廣告的[轉換率](conversion-rate.md)，是行銷的核心戰場
- 跟 [SEO](seo.md)、[A/B Testing](ab-testing.md) 都有密切關係
- 設計師和工程師會需要知道這是一個獨立的頁面，不是改首頁
