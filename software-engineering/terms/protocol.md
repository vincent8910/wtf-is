# Protocol（通訊協定）— 電腦之間溝通的規矩

> **白話說：** Protocol 就像開會的議事規則——先舉手、再發言、一次一個人講。大家都遵守同一套規矩，溝通才不會亂成一團。

---

## 它到底是什麼？

**Protocol（通訊協定）** 就是電腦之間溝通時，雙方共同遵守的規則。

想像兩個人要溝通：

```
沒有 Protocol（沒有規矩）：
  A 講中文、B 講日文 → 雞同鴨講
  A 用寫信、B 打電話 → 溝通管道不同
  A 覺得溝通完了、B 還在等回覆 → 搞不清楚狀態

有 Protocol（有規矩）：
  規定：一律用中文
  規定：一律用 Email
  規定：收到要回覆「已收到」
  → 雙方都照著做，溝通順暢 ✅
```

電腦的世界裡有非常多 Protocol，各管不同的事：

| Protocol | 負責什麼 | 生活比喻 |
|----------|---------|---------|
| [HTTP/HTTPS](http-https.md) | 瀏覽網頁 | 圖書館的借書流程 |
| TCP | 確保資料完整送達 | 掛號信（保證收到） |
| UDP | 快速傳輸，不保證完整 | 丟傳單（快但可能飛走） |
| FTP | 傳檔案 | 快遞寄包裹 |
| SMTP | 寄電子郵件 | 郵局寄信流程 |
| [WebSocket](websocket.md) | 即時雙向溝通 | 打電話 |

## 生活比喻 / 實際例子

Protocol 在日常生活中到處都有：

| 日常規矩 | 對應的概念 |
|---------|----------|
| 靠右行駛 | 交通 Protocol——大家遵守才不會撞車 |
| 打電話先說「喂」 | 通話 Protocol——雙方確認連線 |
| 寄信要寫地址、貼郵票 | 郵件 Protocol——格式不對寄不到 |
| 麥當勞點餐：先排隊、再點餐、再付錢、再等叫號 | 服務 Protocol——流程不對會混亂 |

你每天上網的時候，瀏覽器網址列最前面的 `https://` 就是在告訴你：「我正在用 HTTPS 這套 Protocol 跟伺服器溝通。」

**造句：**「這兩台伺服器用 TCP **Protocol** 溝通，確保每一筆交易資料都完整送達，不會掉資料。」

## 為什麼要知道這個詞？

- 網路世界的一切運作都靠 Protocol——沒有共同規則，電腦之間根本無法溝通
- 你常聽到的 HTTP、HTTPS、TCP、FTP 這些縮寫，全部都是不同的 Protocol
- 當工程師說「這個 Protocol 不支援」，意思是「雙方說的語言不一樣，沒辦法溝通」
- [MCP](mcp.md)（Model Context Protocol）也是一種 Protocol——它定義了 AI 和外部工具之間的溝通規則
