# JSON — 軟體之間傳資料用的標準格式

> **白話說：** 就像表格有固定的欄位格式，JSON 是一種「大家都看得懂」的資料寫法，讓不同系統能互相溝通。

---

## 它到底是什麼？

假設你要把一筆客戶資料從 A 系統傳到 B 系統。如果 A 用自己的格式寫、B 也用自己的格式寫，兩邊根本對不起來。

**JSON**（JavaScript Object Notation）就是大家約定好的「共同格式」。它用很簡單的方式把資料列出來，長這樣：

```json
{
  "姓名": "小明",
  "年齡": 28,
  "會員": true,
  "興趣": ["看電影", "打籃球"]
}
```

看得出來嗎？就是用大括號把資料包起來，每一筆都是「名稱：值」的組合。人看得懂，電腦也看得懂。

## 生活比喻 / 實際例子

- **統一發票的格式**：不管哪家店開的發票，欄位都是一樣的（日期、品項、金額），所以國稅局的系統能看懂所有店家的資料
- **履歷表範本**：大家用同樣的表格格式寫履歷，HR 才能快速對照比較
- **快遞包裹上的標準標籤**：寄件人、收件人、地址、電話，每家物流都用同樣格式，所以才能互通

### 實際使用場景

- 前端跟後端透過 [API](api.md) 傳資料，幾乎都用 JSON 格式
- 你在網站上送出表單（姓名、Email、電話），這些資料會被包成 JSON 傳給 [Server](server.md)
- 很多設定檔（像 `package.json`）也是 JSON 格式

## 為什麼要知道這個詞？

- 「API 回傳的是 JSON 格式」→ 後端送回來的資料是用 JSON 這種格式寫的
- 「幫我看一下這個 JSON」→ 要你看一段結構化的資料內容
- 「JSON 格式不對」→ 資料的寫法有錯（可能少了逗號、括號沒對上），系統讀不懂
