# CI/CD — 迴轉壽司的自動化流水線

> **白話說：** 程式碼寫完後自動測試、自動部署的流水線，不用人工手動搬。

---

## 它到底是什麼？

**CI/CD** 是兩個概念的合稱：

- **CI（Continuous Integration，持續整合）**：工程師寫完程式碼，系統自動幫你跑測試、檢查有沒有問題
- **CD（Continuous Deployment，持續部署）**：測試通過後，系統自動把程式碼[部署](deploy.md)到正式環境

想像一家迴轉壽司——

```
沒有 CI/CD（傳統餐廳）：
  師傅做好壽司 → 喊一聲 → 服務生走過來拿 → 送到客人桌上
  （手動、慢、可能送錯桌）

有 CI/CD（迴轉壽司）：
  師傅做好壽司 → 放上輸送帶 → 自動送到客人面前
  （自動化、快速、不會出錯）
```

**CI/CD 就是壽司的輸送帶——做好了自動送到客人面前。**

## 生活比喻 / 實際例子

也像工廠的品管產線——產品做好後自動經過品檢站（CI：自動測試），合格的自動包裝出貨（CD：自動部署），不合格的被退回去（測試失敗，不讓上線）。

### 常見情境

- **工程師：**「CI 跑失敗了，有一個測試沒過。」→ 自動化品管發現了問題
- **PM：**「推上去之後多久能上線？」→ 「CI/CD 跑完大概 10 分鐘。」
- **工程師：**「我們的 CI/CD [pipeline](pipeline.md)（流水線）要加一個步驟。」→ 流水線要多加一個關卡

## 為什麼要知道這個詞？

CI/CD 決定了「工程師寫完程式碼到用戶能用到新功能」之間要多久。有好的 CI/CD，一天可以部署好幾次；沒有的話，每次上線都是大工程。當 PM 問「什麼時候能上線」，CI/CD 的效率是關鍵因素之一。
