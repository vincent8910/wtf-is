# Algorithm（演算法）— 解決問題的 SOP

> **白話說：** 演算法就是一份食譜——按照步驟做，就能從食材變成一道菜。不同的食譜（演算法）做出來的菜味道不同、花的時間也不同。

---

## 它到底是什麼？

**Algorithm（演算法）** 聽起來很厲害，但其實就是：

> **一套「按照步驟解決問題」的方法。**

你每天都在用演算法，只是不知道：

```
泡泡麵的演算法：
  1. 打開碗麵蓋子
  2. 撕開調味包倒進去
  3. 加熱水到線
  4. 蓋上蓋子等 3 分鐘
  5. 打開吃
  → 完成！
```

電腦裡的演算法也是一樣——給它一個問題和一些資料，它按照步驟處理，最後給你答案。

差別在於：有些演算法很快（微波爐加熱），有些很慢（用炭火慢烤），解決同一個問題可以有很多不同的演算法，工程師要選最適合的那個。

## 生活比喻 / 實際例子

你每天都在「被演算法服務」：

| 你做的事 | 背後的演算法在做什麼 |
|---------|------------------|
| Google 搜尋 | 從幾十億個網頁中，用演算法找出最相關的排在前面 |
| YouTube 推薦影片 | 分析你看過什麼，用演算法猜你想看什麼 |
| Google Maps 導航 | 用演算法算出最快的路線 |
| 蝦皮排序「價格低到高」 | 用排序演算法把商品重新排列 |
| 抖音 / IG 動態排序 | 用演算法決定你先看到誰的貼文 |

演算法不是什麼神秘的東西，就是「做事的步驟和方法」。但好的演算法和差的演算法差很多——就像 Google Maps 可以幫你找到最快的路，但如果演算法很差，可能會叫你繞整個台灣一圈才到隔壁。

**造句：**「YouTube 的推薦**演算法**改了，最近一直推給我奇怪的影片。」

## 為什麼要知道這個詞？

- 「演算法」是科技新聞裡出現頻率最高的術語之一——「IG 演算法改版」、「YouTube 演算法推薦」
- 大家口中的「被演算法控制」，意思是社群平台用演算法決定你看到什麼內容
- 工程師面試最常考的就是演算法——因為它代表「解決問題的能力」
- 當有人說「這個演算法的時間複雜度太高」，白話就是「這個做法太慢了，要換一種更快的方法」
