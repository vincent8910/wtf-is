# TypeScript — JavaScript 的加強版，幫你提前抓錯

> **白話說：** TypeScript 就像有自動糾正功能的文書處理器——你打錯字的時候立刻畫紅線提醒你，不用等到交出去才被退件。

---

## 它到底是什麼？

[JavaScript](javascript.md) 是網頁的程式語言，全世界的網站幾乎都在用。但它有個問題：**太自由了，容易出錯卻不告訴你。**

想像你在寫一封信：

```
用 JavaScript 寫信（沒有拼字檢查）：
  你寫了「台北是 101」→ 沒人提醒你 → 寄出去 → 對方看到才發現怪怪的

用 TypeScript 寫信（有拼字檢查＋文法檢查）：
  你寫了「台北是 101」→ 立刻畫紅線：「你是不是要寫『台北有 101』？」
  → 當場修正 → 寄出去 → 沒問題
```

**TypeScript = JavaScript + 型別檢查。**

「型別」就是告訴電腦：這個東西是數字、那個東西是文字、另一個是日期。有了這些標記，電腦就能在你寫程式的時候就幫你抓到「你把文字當數字在算」之類的低級錯誤。

## 生活比喻 / 實際例子

就像填表格：

| | JavaScript（自由填寫） | TypeScript（有格式限制的表格） |
|---|---|---|
| 電話欄位 | 隨便填，你寫「不告訴你」也行 | 限定只能填數字，填錯馬上提醒 |
| 日期欄位 | 你寫「上禮拜」也行 | 限定 YYYY/MM/DD 格式 |
| 結果 | 表格收回來一堆亂七八糟的資料 | 資料格式整齊、後續處理不出錯 |

TypeScript 就是幫 JavaScript 加上這些「欄位格式限制」，讓程式碼更嚴謹。

**造句：**「我們新專案改用 **TypeScript** 了，雖然一開始寫起來多一點工，但 [Bug](bug-debug.md) 真的少很多。」

## 為什麼要知道這個詞？

- 現在越來越多公司的新專案都用 TypeScript，已經快變成業界標準了
- TypeScript 最後還是會被轉換成 JavaScript 才能在瀏覽器跑，所以它是 JavaScript 的「超集合」
- 當工程師說「我們用 TS 寫」，TS 就是 TypeScript 的縮寫
- 選擇用 TypeScript 還是 JavaScript，是專案初期常見的技術決策話題
