# SQL — 跟資料庫溝通的語言

> **白話說：** [資料庫](database.md)就像一個超大的 Excel，而 SQL 就是你「跟這個 Excel 下指令」的方式——幫我找某筆資料、新增一筆、刪掉一筆。

---

## 它到底是什麼？

SQL 全名是 Structured Query Language（結構化查詢語言）。

它是專門用來**跟[資料庫](database.md)溝通的語言**。你想從資料庫裡找資料、新增資料、修改資料、刪除資料，都要用 SQL 來「告訴」資料庫你要做什麼。

舉例來說：
- 「幫我找出所有台北市的客戶」→ SQL 查詢
- 「把這個會員的 Email 改掉」→ SQL 更新
- 「刪掉三個月前的測試資料」→ SQL 刪除

SQL 是最老牌也最普遍的資料庫語言，幾乎所有的傳統[資料庫](database.md)（MySQL、PostgreSQL、SQL Server）都用它。

## 生活比喻 / 實際例子

**圖書館的查詢系統：**
- 圖書館有幾十萬本書（資料庫裡的資料）
- 你走到查詢電腦，輸入「作者：村上春樹，出版年：2020 以後」→ 這就像一段 SQL
- 電腦幫你從幾十萬本書裡找出符合條件的那幾本

**你什麼時候會聽到：**
- 「我寫一段 **SQL** 去撈資料，看一下那個數字對不對。」
- 「這個報表要下 **SQL** query 才能拉出來。」
- 「不要直接在 production 上跑 **SQL**，太危險了！」

## 為什麼要知道這個詞？

- 當你跟工程師或資料分析師要某些數據，他們就是在寫 SQL 去查
- 有些公司的 PM 或行銷也會學基本的 SQL，自己撈資料做分析
- 工程師說「下一段 SQL」或「跑一個 query」，就是在跟資料庫提問
- 如果工程師說「SQL injection」，那是一種[資安漏洞](vulnerability.md)——駭客透過輸入惡意的 SQL 來偷取資料
