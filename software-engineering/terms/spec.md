# Spec（規格）— 蓋房子前的藍圖

> **白話說：** Spec 就像建築藍圖——沒有它，工程師不知道要蓋什麼。

---

## 它到底是什麼？

**Spec** 是 **Specification** 的縮寫，中文叫「規格書」。

```
你跟建築師說：「我要一間房子。」
建築師：「⋯幾層？幾間房？要不要車庫？廁所幾間？」

你把這些答案寫下來，就是一份 spec。
```

軟體也一樣。你說「我要一個登入功能」，工程師會問：

- 可以用 Google 帳號登入嗎？
- 密碼最少幾個字？
- 輸入錯 5 次要不要鎖住？

這些答案全部寫下來，就是一份 **spec**。

## 一份好的 spec 要包含什麼？

| 項目 | 蓋房子的比喻 |
|------|-------------|
| **功能描述** — 做什麼？ | 我要蓋兩層樓住宅 |
| **使用者是誰** | 住的人是四口之家 |
| **預期行為** — 操作後發生什麼？ | 按門鈴 → 對講機響 → 可以開門 |
| **邊界條件** — 異常怎麼辦？ | 停電時門鎖要能手動打開 |
| **不做什麼** | 不蓋游泳池、不蓋地下室 |

最後一項「不做什麼」可以避免 [scope creep（範圍蔓延）](scope.md)。

## 日常場景

- **「spec 寫好了嗎？沒有 spec 沒辦法開工」**
  → 藍圖還沒好，不知道蓋什麼
- **「這個 spec 太模糊了」**
  → 你只寫了「我要一間房子」，沒說幾層幾房
- **「這個不在 spec 裡面」**
  → 藍圖上沒畫游泳池，所以沒蓋

> 💡 身為 PM，**你通常就是寫 spec 的那個人**。與其花三天寫一份完美的 spec，不如先寫一份 80 分的讓團隊 review，再快速修改。
