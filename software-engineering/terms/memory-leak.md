# Memory Leak（記憶體洩漏）— 借書不還的人

> **白話說：** Memory Leak 就像有人一直跟圖書館借書卻從不還——書架上的書越來越少，最後整間圖書館沒書可借，只能暫時關門。

---

## 它到底是什麼？

電腦的**記憶體（Memory）**是有限的資源，程式在運作的時候會跟系統「借」一塊記憶體來用。正常情況下，用完應該要「還回去」，讓其他程式也能用。

**Memory Leak（記憶體洩漏）** 就是程式借了記憶體卻忘了還：

```
正常的程式：
  借記憶體 → 用 → 用完歸還 → 記憶體空出來給別人用 ✅

有 Memory Leak 的程式：
  借記憶體 → 用 → 忘了還 → 再借 → 又忘了還 → 再借...
  → 記憶體越借越多 → 系統越來越慢 → 最後整個當掉 💥
```

最可怕的是，Memory Leak 通常不會立刻出事。它像慢性病——程式剛啟動時好好的，跑了幾小時或幾天後突然越來越慢，最後 [Crash](crash.md)。

## 生活比喻 / 實際例子

除了借書不還，再舉幾個生活例子：

| Memory Leak 比喻 | 說明 |
|-----------------|------|
| 浴缸放水忘了關 | 水（記憶體）一直流，遲早溢出來 |
| 桌子上的碗盤不收 | 吃完不洗碗、碗越堆越多，最後沒地方吃飯 |
| 手機一直開 App 不關 | 背景程式越開越多，手機越來越卡 |

你手機用久了會越來越慢，重開機就恢復正常——就是因為重開機把那些「借了沒還」的記憶體全部清乾淨了。

**造句：**「這個 App 用了半小時就開始卡頓，八成是有 **Memory Leak**，要查一下是哪裡佔著記憶體不放。」

## 為什麼要知道這個詞？

- 系統越跑越慢、放著放著就當掉，Memory Leak 是最常見的原因之一
- 手機 App 特別容易出現——畫面切來切去如果沒處理好，記憶體就會一直累積
- 工程師常用監控工具觀察記憶體用量，如果看到一條「只漲不跌」的線，通常就是 Memory Leak
- 最簡單的暫時解法就是重啟程式（就像手機重開機），但根本解法是找到程式碼裡「忘了歸還」的那段
