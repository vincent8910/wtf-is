# Version / SemVer（語意化版本）— 軟體的身分證號碼

> **白話說：** SemVer 就像手機系統的版本號——iOS 16.2 升到 iOS 17.0，光看數字就知道是大更新還是小修補。

---

## 它到底是什麼？

你一定看過這種數字：`v2.1.3`、`iOS 17.0.1`、`Chrome 120.0.6099`。

這些就是**版本號（Version Number）**，告訴你「這個軟體現在是第幾版」。

**SemVer**（Semantic Versioning，語意化版本）是目前最常用的版本號規則，格式是：

```
主版號.次版號.修訂號
 Major . Minor . Patch

例如 v2.1.3：
  2 = 主版號（第 2 代大改版）
  1 = 次版號（新增了 1 個功能）
  3 = 修訂號（修了 3 次小 Bug）
```

用手機來理解：

| 版本變化 | 意思 | 手機比喻 |
|---------|------|---------|
| 1.0.0 → **2**.0.0 | 主版號升級：大改版，可能不相容 | iPhone 14 → iPhone 15（整支換新） |
| 2.0.0 → 2.**1**.0 | 次版號升級：新增功能，向下相容 | iOS 17.0 → iOS 17.1（多了新功能） |
| 2.1.0 → 2.1.**1** | 修訂號升級：修 Bug，沒加新東西 | iOS 17.1 → iOS 17.1.1（修了一個問題） |

## 生活比喻 / 實際例子

想像你開一間便當店：

```
便當 v1.0.0 — 基本款：白飯 + 雞腿 + 青菜

便當 v1.1.0 — 加了配菜（新功能）：白飯 + 雞腿 + 青菜 + 滷蛋
便當 v1.1.1 — 修正問題：把太鹹的青菜換掉（Bug Fix）
便當 v1.2.0 — 多了湯品（又一個新功能）

便當 v2.0.0 — 大改版：換成健康餐盒（整個概念不同了！）
  → 舊的便當盒不能用了（不向下相容）
```

**造句：**「這次我們升到 **v3.0**，是破壞性更新，舊版的 [API](api.md) 會不能用，要通知客戶升級。」

## 為什麼要知道這個詞？

- 看到版本號就能判斷更新幅度——主版號跳了表示大改，修訂號加 1 表示只是小修
- 工程師說「這是 breaking change，要升 Major」，意思是這次改動會讓舊版不相容
- [npm](npm.md) 等套件管理工具靠 SemVer 判斷要不要自動升級——`^2.1.0` 表示「2.x.x 都可以自動升級，但不要升到 3.0」
- 產品經理也常用版本號跟客戶溝通：「下個大版本會加入 AI 功能」
