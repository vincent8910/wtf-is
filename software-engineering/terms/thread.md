# Thread（執行緒）— 餐廳裡的多位廚師

> **白話說：** Thread 就像餐廳裡有好幾個廚師同時做不同的菜——一個在煎牛排、一個在煮湯、一個在做甜點，大家同時工作，出餐速度飛快。

---

## 它到底是什麼？

**Thread（執行緒）** 是程式裡面「同時做多件事」的最小單位。

想像你一個人在廚房煮晚餐：

```
單執行緒（一個廚師）：
  煮飯（等 20 分鐘）→ 炒菜（10 分鐘）→ 煮湯（15 分鐘）
  → 總共要 45 分鐘

多執行緒（三個廚師）：
  廚師 A：煮飯（20 分鐘）
  廚師 B：同時炒菜（10 分鐘）
  廚師 C：同時煮湯（15 分鐘）
  → 總共只要 20 分鐘！
```

電腦裡的 Thread 就是這些廚師。一個程式可以開好幾個 Thread，讓它們同時處理不同的任務，大大加快速度。

但多執行緒也有風險——就像兩個廚師同時要用同一口鍋，得有人協調，不然會撞在一起。程式裡的 Thread 如果沒處理好「共用資源」的問題，就會出 [Bug](bug-debug.md)。

## 生活比喻 / 實際例子

| 場景 | 單執行緒 | 多執行緒 |
|-----|---------|---------|
| 夜市攤位 | 老闆一個人切、煎、裝盤、收錢 | 老闆煎，老闆娘裝盤收錢 |
| 便利商店 | 只開一台收銀機 | 三台收銀機同時結帳 |
| 客服中心 | 只有一位客服，一次接一通電話 | 十位客服同時接十通電話 |

你打開工作管理員（Windows）或活動監視器（Mac），會看到每個程式用了幾個 Thread。Chrome 瀏覽器特別愛開 Thread——每個分頁都是獨立的，所以一個分頁當掉不會影響其他的。

**造句：**「這個下載器開了 8 個 **Thread** 同時下載，所以速度比單線程快好幾倍。」

## 為什麼要知道這個詞？

- 電腦的 CPU 核心數決定了能同時跑多少個 Thread——這就是為什麼「8 核心處理器」比「4 核心」快
- 遊戲、影片編輯、伺服器處理大量請求，都很依賴多執行緒
- 跟 [Concurrency（並發）](concurrency.md) 是兄弟概念——Concurrency 是「同時處理多件事」，Thread 是「用什麼方式來同時處理」
- 當你聽到「這個程式支援多執行緒」，意思就是它能同時做好幾件事，效率更高
