# Redis — 速度超快的「記憶體資料庫」，專門用來做快取

> **白話說：** 就像收銀台旁邊的小架子——把最常被拿的商品放在最順手的地方，不用每次都跑去倉庫拿。

---

## 它到底是什麼？

Redis 是一種把資料存在記憶體（RAM）裡的資料庫。跟一般[資料庫](database.md)（存在硬碟裡）相比，Redis 的讀寫速度快非常多，因為記憶體的存取速度比硬碟快幾百倍。

但記憶體空間有限又比較貴，所以 Redis 通常不拿來存所有資料，而是存「最常被查詢的資料」來加速——也就是當作[快取（Cache）](cache.md)來用。

Redis 常見的用途：
- **快取**：把熱門商品資訊、使用者 Session 存在 Redis，不用每次都去查資料庫
- **排行榜**：遊戲排行榜、即時數據排名
- **計數器**：按讚數、瀏覽次數
- **佇列（Queue）**：排隊等處理的任務

## 生活比喻 / 實際例子

想像一家便利商店：

- **資料庫（MySQL / MongoDB）** = 後面的大倉庫，什麼都有，但跑去拿要花時間
- **Redis** = 收銀台旁邊的小架子，最熱賣的口香糖和飲料放這裡，伸手就拿到

顧客問「有沒有可樂？」——你不用跑去倉庫，架子上直接有。

實際例子：
- 工程師說：「首頁的商品列表加了 **Redis** 快取，讀取速度快了 10 倍」
- 工程師說：「用戶的 [Session](session.md) 存在 **Redis** 裡」
- PM 問：「為什麼排行榜更新這麼快？」工程師：「因為排行榜的資料放在 **Redis**」

## 為什麼要知道這個詞？

- 幾乎所有有一定規模的網站都會用 Redis 來加速
- 當工程師說「加個 Redis」，通常是在解決「速度太慢」的問題
- Redis 常跟 [MongoDB](mongodb.md) 或其他資料庫搭配使用，各負責不同的事
