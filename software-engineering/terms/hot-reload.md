# Hot Reload（熱重載）— 邊裝潢邊住

> **白話說：** Hot Reload 就像邊裝潢邊住——油漆工刷完客廳那面牆，你站在旁邊立刻就看到新顏色，不用搬出去再搬回來。

---

## 它到底是什麼？

以前工程師改程式碼的流程是這樣的：

```
沒有 Hot Reload：
  改一行程式碼
  → 存檔
  → 停止程式
  → 重新啟動程式（等 10 秒 ~ 幾分鐘）
  → 重新打開頁面
  → 找到剛才在看的地方
  → 終於看到改動的效果
  → 發現顏色不對，再來一次...😩
```

每改一次就要等一次，一天改幾十次，光是等待就浪費大把時間。

```
有 Hot Reload：
  改一行程式碼
  → 存檔
  → 畫面立刻自動更新！（不到 1 秒）
  → 馬上看到效果 ✅
  → 不對？再改，立刻又更新
  → 開發效率翻倍 🚀
```

**Hot Reload 讓你改了程式碼之後，不用重新啟動整個程式，畫面就自動反映最新的修改。**

更厲害的是，有些 Hot Reload 甚至會**保留你目前的操作狀態**——比如你在一個填了一半的表單上改了按鈕顏色，Hot Reload 後表單的內容還在，不用重新填。

## 生活比喻 / 實際例子

| | 沒有 Hot Reload | 有 Hot Reload |
|---|---|---|
| 裝潢比喻 | 改客廳牆色 → 全家搬出去 → 油漆 → 搬回來 → 才看到 | 油漆工直接刷 → 你站旁邊馬上看到 |
| 寫報告比喻 | 改一個字 → 重新列印整份 → 翻到那一頁看 | 改一個字 → 螢幕上立刻更新 |
| 試衣服比喻 | 換一件衣服 → 先回家 → 再去試衣間 | 在試衣間直接一件一件換著看 |

你在用的很多開發工具都支援 Hot Reload：

| 工具 / 框架 | Hot Reload 體驗 |
|------------|----------------|
| React（前端框架） | 改 [CSS](css.md) 或元件，瀏覽器立刻更新 |
| Flutter（App 框架） | 改完程式碼，手機畫面秒變 |
| Next.js | 存檔後網頁自動刷新 |
| VS Code Live Server | 改 [HTML](html.md) 存檔，瀏覽器自動重新整理 |

**造句：**「我們用的框架支援 **Hot Reload**，所以調 UI 超快，改完存檔馬上就看到效果。」

## 為什麼要知道這個詞？

- Hot Reload 大幅提升開發效率，是現代前端開發的標配功能
- 當工程師說「存檔就能看到效果」，背後就是 Hot Reload 在運作
- Hot Reload 跟「自動重啟」不太一樣——Hot Reload 是「局部更新」，自動重啟是「整個砍掉重來」
- 有些複雜的改動還是需要手動重啟——Hot Reload 不是萬能的，但 80% 的情況都夠用
