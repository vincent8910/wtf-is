# DNS — 網址的電話簿

> **白話說：** 你輸入 `google.com`，DNS 幫你查出 Google 的伺服器在哪裡，就像在電話簿裡查名字找到電話號碼一樣。

---

## 它到底是什麼？

DNS 全名是 Domain Name System（網域名稱系統）。

電腦之間其實是用一串數字（[IP 位址](ip-address.md)）來互相辨識的，像是 `142.250.185.14`。但你不可能記住每個網站的數字吧？

所以 DNS 的功能就是：**把你輸入的網址（像是 `google.com`）轉換成電腦看得懂的 IP 位址**。就像電話簿把「王小明」對應到「0912-345-678」一樣。

## 生活比喻 / 實際例子

**電話簿 / 手機通訊錄：**
- 你不需要記住朋友的電話號碼，只要在通訊錄裡找到名字就能打電話
- DNS 就是網路世界的通訊錄——你輸入好記的網址，它幫你找到實際的伺服器位置

**你什麼時候會聽到：**
- 「[網域](domain.md)買好了，但 **DNS** 還沒設定，所以網站還打不開。」
- 「**DNS** 生效要等幾個小時，不會馬上有效。」（這叫 DNS propagation）
- 「查一下 **DNS** 指到哪裡，可能指錯伺服器了。」

## 為什麼要知道這個詞？

- 當你買了一個[網域](domain.md)（像 `mycompany.com`），需要設定 DNS 才能讓這個網址連到你的網站
- 網站搬家時，工程師會說「改 DNS 指向」——意思是把網址指到新的[伺服器](server.md)
- DNS 設定改了之後不會馬上生效，通常要等幾小時到一天，這段時間網站可能時好時壞是正常的
- 如果網站突然打不開，工程師可能會先檢查「是不是 DNS 的問題」
