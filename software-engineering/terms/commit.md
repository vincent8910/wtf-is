# Commit（提交） — 對程式碼按一次「存檔」

> **白話說：** 就像遊戲裡的「存檔點」，把你目前改的東西記錄下來，之後隨時可以回到這個點。

---

## 它到底是什麼？

你在寫 Word 文件時，會習慣按 Ctrl+S 存檔。**Commit** 就是程式碼版本的存檔，但比 Ctrl+S 更厲害——它會幫你記住「你這次改了什麼」和「你為什麼改」。

每次 commit 都像是在時間軸上釘了一個圖釘：

- 「11:30 — 修好了登入頁面的錯字」
- 「14:00 — 加上忘記密碼功能」
- 「16:45 — 調整手機版的排版」

如果後來發現某次修改弄壞了東西，可以直接跳回之前的存檔點。

## 生活比喻 / 實際例子

- **遊戲存檔**：打 Boss 之前先存檔，打輸了可以讀檔重來
- **拍照記錄裝修進度**：每做完一個階段就拍張照，有問題可以回頭看是哪個階段出的差錯
- **記帳 App 的每筆紀錄**：每筆消費都記下時間、金額、用途，之後才查得到

### 一個 Commit 長什麼樣？

工程師做完一組修改後，會寫一段簡短的訊息（叫 commit message），說明這次改了什麼：

```
feat: 新增忘記密碼功能
fix: 修正手機版登入按鈕跑版
docs: 更新 API 文件
```

這些訊息讓團隊所有人都能快速看懂每次修改的目的。

## 為什麼要知道這個詞？

- 「我已經 commit 了」→ 代表工程師已經把修改存檔到版本紀錄裡
- 「你那個 commit 的內容是什麼？」→ 在問你這次存檔改了哪些東西
- 「這個 bug 是哪個 commit 引入的？」→ 在追查是哪次修改造成了問題
- 「幫我看一下 commit log」→ 要看修改的歷史紀錄
