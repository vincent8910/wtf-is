# Regression（回歸測試）— 修水管後檢查隔壁馬桶

> **白話說：** 確認新功能或修改沒有弄壞原本正常的功能。

---

## 它到底是什麼？

**Regression**（回歸測試）就是每次改了什麼東西之後，回頭把舊功能也測一遍，確認它們還是正常的。

想像你家請水電師傅來修廚房水管——

```
師傅修好廚房水管：「好了，廚房不漏了。」
你（回歸測試）：去檢查一下⋯⋯
  ☐ 浴室水龍頭還正常嗎？→ ✅ 正常
  ☐ 馬桶沖水正常嗎？→ ❌ 糟了，水壓變小了！
  ☐ 洗衣機進水正常嗎？→ ✅ 正常
```

**Regression 就是修完水管後，去隔壁檢查馬桶有沒有壞掉。**

## 生活比喻 / 實際例子

軟體系統的程式碼是互相連動的——改了 A 功能，可能不小心影響到 B 功能。就像拉了一條水管，結果另一邊的水壓變了。

### 常見情境

- **QA（品質測試人員）：**「新功能測完了，現在跑 regression，確認舊功能沒壞。」
- **PM：**「這次改動影響範圍大嗎？要不要做完整的 regression？」→ 改動很大的話，要把所有功能都重新測一遍
- **工程師：**「regression 發現一個 [bug](bug-debug.md)（程式錯誤），登入頁面壞了。」→ 回頭一測，發現原本好的登入功能壞了

## 為什麼要知道這個詞？

每次軟體更新，最怕的就是「修好了新問題，弄壞了舊功能」。Regression 就是防止這件事發生的安全網。當你聽到「regression 沒過」，代表新的修改把舊功能搞壞了，需要先修好才能上線。
