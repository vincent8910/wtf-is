# Branch（分支） — 在不影響主版本的情況下，開一條平行路線去做事

> **白話說：** 就像開了一個「平行宇宙」，你在裡面怎麼改都不會影響正式版本，改好了再合併回來。

---

## 它到底是什麼？

想像你正在寫一份企劃書，已經有一版大家看過的定稿了。現在你想改幾段文字試試看，但又怕改壞，你會怎麼做？

你會「另存新檔」一份副本，在副本上改。改好了、大家覺得 OK，再把修改的部分合併回去。

在軟體開發裡，**Branch（分支）** 就是這個「另存新檔」的概念。工程師開一個分支，在上面寫新功能或修問題，寫完後再合併回主版本。

## 生活比喻 / 實際例子

- **平行宇宙**：你在平行宇宙裡試新髮型，覺得好看再帶回現實世界
- **Google 文件的「建議編輯」模式**：你改了東西，但原本的內容不會動，等確認後才「接受變更」
- **便利商店的試吃**：先小規模試做一批新口味（分支），客人反應好再正式上架（合併回主線）

### 工程師在做什麼？

假設現在網站要加一個「忘記密碼」功能：

1. 工程師從主版本（`main`）開一個分支，通常取名叫 `feature/forgot-password`
2. 在這個分支上寫程式、測試，怎麼改都不會影響線上的網站
3. 寫完後發 [PR（Pull Request）](pull-request.md)，請同事看過
4. 確認沒問題，把分支合併（[Merge](merge.md)）回主版本
5. 刪掉這個已完成的分支

## 為什麼要知道這個詞？

- 工程師說「我開了一個 branch 在做」→ 意思是他正在一個獨立區域開發，不會影響現有功能
- 「你的 branch 名稱是什麼？」→ 在問你在哪條路線上工作（團隊要追蹤進度）
- 「先不要合併那個 branch」→ 表示那個功能還沒準備好，先別上線
