# WebView — App 裡面的迷你瀏覽器

> **白話說：** WebView 就像在 LINE 裡面直接打開一個網頁——你不用跳出去用 Safari 或 Chrome，在 App 裡面就能看網頁內容。

---

## 它到底是什麼？

**WebView** 就是在 App 裡面嵌入一個小型瀏覽器，讓用戶不用離開 App 就能瀏覽網頁。

你一定有過這種經驗：

```
在 LINE 裡收到一個網址連結
  → 點下去
  → 網頁直接在 LINE 裡面打開了！
  → 你沒有跳到 Safari 或 Chrome
  → 看完按叉叉，回到 LINE 聊天畫面
```

那個「在 LINE 裡面開的網頁」就是透過 WebView 實現的。

對使用者來說，感覺像是 App 的一部分；但其實那個畫面就是一個網頁，跟你用瀏覽器打開是一樣的東西。

## 生活比喻 / 實際例子

WebView 就像百貨公司裡面開的美食街：

```
沒有 WebView：
  你在百貨公司逛街 → 想吃飯 → 走出百貨公司 → 去外面的餐廳 → 吃完再走回來

有 WebView：
  你在百貨公司逛街 → 想吃飯 → 直接上美食街 → 吃完繼續逛
  → 不用離開百貨公司
```

你日常已經在用的 WebView：

| App | 什麼時候用到 WebView |
|-----|-------------------|
| LINE | 點連結開網頁、LINE Pay 的活動頁面 |
| Facebook | 點朋友分享的新聞連結 |
| 蝦皮 | 某些活動頁面、優惠券頁面 |
| 各種銀行 App | 信用卡優惠頁面、活動說明 |

**造句：**「這個活動頁面用 **WebView** 嵌在 App 裡就好，不用特地寫原生頁面，改版也比較快。」

## 為什麼要知道這個詞？

- 很多 App 的部分功能其實是用 WebView 載入網頁做的——因為網頁改版快、不用重新送審 [App Store](app-review.md)
- 但 WebView 的效能和使用體驗通常不如原生畫面——滑起來可能比較卡、互動比較慢
- 當 PM 說「這個頁面用 WebView 做」，意思是「做成網頁，然後嵌進 App 裡面」
- 有些 App 其實整個都是 WebView 包出來的（叫做 Hybrid App），外面看是 App，裡面全都是網頁
