# DevOps — 讓程式碼從「寫好」到「上線」這段路更順暢的角色和文化

> **白話說：** 就像工廠裡負責維護生產線的人——工人（工程師）負責做產品，DevOps 負責確保生產線跑得順、東西能自動包裝出貨。

---

## 它到底是什麼？

DevOps 是 **Development（開發）** + **Operations（維運）** 兩個字合在一起。

傳統做法是：工程師寫完程式碼後，丟給另一組「維運人員」去部署和維護[伺服器](server.md)。兩邊各做各的，溝通常出問題。

DevOps 的理念是：打破這個隔閡，讓開發和維運緊密合作（甚至同一個人做），透過自動化工具讓「寫好程式碼 → 測試 → [部署](deploy.md) → 上線」這整條路變得又快又穩。

DevOps 工程師通常負責的事情：
- 設定 [CI/CD](ci-cd.md) 自動化流程
- 管理[雲端](cloud.md)伺服器和基礎設施
- 用 [Docker](container-docker.md) 打包應用程式
- 設定[監控和告警](monitor-alert.md)系統
- 確保系統穩定、自動擴展

## 生活比喻 / 實際例子

想像一家餐廳：

- **廚師（開發工程師）** = 負責研發新菜色、把菜做好
- **DevOps** = 負責維護廚房設備、確保瓦斯管線正常、食材自動補貨、出菜動線順暢

廚師專心做菜就好，DevOps 確保整個廚房的「基礎設施」運作正常。

實際例子：
- 主管說：「我們需要找一個 **DevOps**，現在部署都是手動的，太慢了」
- 工程師說：「這個要問 **DevOps**，他負責管伺服器」
- PM 問：「新版本多久能上線？」工程師：「**DevOps** 把 CI/CD 建好後，以後每次改完就能自動部署」

## 為什麼要知道這個詞？

- 招人時常看到「DevOps Engineer」的職缺，你要知道他們在做什麼
- 當團隊討論「部署太慢」「伺服器不穩」「需要自動化」，通常就是 DevOps 的範疇
- README 的「依工程師類型分類」裡有一整個 DevOps 區塊，裡面的詞都跟這個角色相關
