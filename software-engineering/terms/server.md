# Server（伺服器）— 24 小時不打烊的服務生

> **白話說：** Server 就是一台 24 小時開著的電腦，專門等別人來問問題、要資料。

---

## 它到底是什麼？

**Server（伺服器）** 聽起來很厲害，但本質就是一台電腦——只是它**不關機、不休息、24 小時待命**，專門服務別人的請求。

用餐廳比喻：

```
你（客人）    →  服務生（Server）  →  廚房（程式 / 資料庫）
「我要一碗麵」  →   接收點餐        →   開始料理
              ←   端菜上桌        ←   做好了
```

你的手機打開一個 App，按了一個按鈕——背後就是在跟某台 Server 說：「嘿，幫我查一下資料。」Server 收到請求，去 [Database（資料庫）](database.md) 查完，再把結果送回你的手機。

### 實體 vs 雲端

| | 實體伺服器 | 雲端伺服器 |
|---|---|---|
| **比喻** | 自己開餐廳，廚房自己蓋 | 在百貨美食街租攤位 |
| **怎麼用** | 買一台電腦放在機房裡 | 跟 AWS / GCP 租用 |
| **成本** | 前期投資大，自己維護 | 按用量付費，別人維護 |
| **彈性** | 客人暴增時來不及加座位 | 隨時可以加開攤位 |

現在大多數公司都用**雲端伺服器**——不用自己買電腦，像租房子一樣按月付費。

## 日常場景

- **「Server 掛了，網站打不開」**
  → 那台 24 小時服務的電腦出問題了，沒人接客了
- **「要加一台 server 來撐流量」**
  → 客人太多、服務生忙不過來，要多請一個人
- **「server 回應很慢」**
  → 服務生走路太慢，客人等很久
