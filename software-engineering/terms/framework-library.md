# Framework / Library（框架 / 函式庫） — 別人寫好的現成工具，拿來用就好

> **白話說：** Library 是工具箱，你需要什麼拿什麼；Framework 是整套廚房設備，它規定了動線和流程，你按照它的方式來做菜。

---

## 它到底是什麼？

寫程式不用每樣東西都從零開始。很多常見的功能（登入、表單驗證、資料庫連線）前人都寫好了，打包成 **Library（函式庫）** 或 **Framework（框架）** 讓大家直接用。

### Library（函式庫）

像一個**工具箱**。你需要鐵鎚就拿鐵鎚，需要螺絲起子就拿螺絲起子。你決定什麼時候用、怎麼用。

例如：你需要處理日期格式，就裝一個日期處理的 library 來用。

### Framework（框架）

像一間**已經裝潢好的廚房**。它幫你規劃好了動線（食材區、料理區、出餐區），你按照它的流程來做菜。

例如：用了某個框架之後，檔案要放哪裡、程式怎麼寫，框架都有規定好的方式。

## 生活比喻 / 實際例子

| | Library（函式庫） | Framework（框架） |
|--|-------------------|-------------------|
| 比喻 | 工具箱 | 已裝潢好的廚房 |
| 你是主角嗎 | 是，你決定什麼時候用什麼工具 | 框架是主角，你按它的規則來 |
| 彈性 | 高，想怎麼用就怎麼用 | 低一點，但幫你省了很多決定 |
| 例子 | 一個日期處理工具、一個圖表工具 | React、Vue（前端框架）、Django（後端框架） |

### 你可能聽過的框架和函式庫

- **React / Vue / Angular**：前端框架，用來做網頁畫面
- **Express / Django / Spring**：後端框架，用來處理資料和邏輯
- **jQuery / Lodash**：前端常用的函式庫

## 為什麼要知道這個詞？

- 「我們用的框架是 React」→ 團隊選了 React 這套工具來開發前端
- 「有沒有現成的 library 可以用？」→ 在問有沒有別人寫好的工具可以直接拿來用，不用自己從頭寫
- 「這個框架太舊了，要升級」→ 這套基礎工具的版本太老，需要更新
- 「換框架工程很大」→ 因為框架決定了整個開發方式，換框架等於整間廚房重新裝潢
