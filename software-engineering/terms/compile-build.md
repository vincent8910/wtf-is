# Compile / Build（編譯 / 建置）— 把食譜變成一道菜

> **白話說：** Compile 就像把食譜（人寫的程式碼）變成一道真正的料理（電腦能執行的程式）。食譜本身不能吃，要經過烹飪這個步驟才行。

---

## 它到底是什麼？

人寫的程式碼其實是**給人看的**，電腦看不懂。電腦只懂 0 和 1。所以中間需要一個「翻譯 + 加工」的過程：

```
Compile（編譯）：
  你寫的程式碼（食譜）
    ↓ 編譯器翻譯
  電腦能執行的機器碼（料理）

Build（建置）：
  你寫的所有程式碼 + 圖片 + 設定檔（食材 + 食譜 + 調味料）
    ↓ 建置工具打包
  一個完整可以上線的程式（一桌完整的菜）
```

**Compile** 是把程式碼「翻譯」成電腦看得懂的語言。
**Build** 是把整個專案「打包」成可以執行或[部署](deploy.md)的成品。

Build 的範圍比 Compile 大——Build 通常包含了 Compile，加上壓縮檔案、打包資源、處理相依性等一堆步驟。

## 生活比喻 / 實際例子

| 步驟 | 料理比喻 | 軟體比喻 |
|------|---------|---------|
| 原始材料 | 食譜 + 食材 | 程式碼 + 圖片 + 設定檔 |
| 加工過程 | 切菜、烹飪、擺盤 | Compile + 打包 + 壓縮 |
| 最終成品 | 一道可以上桌的料理 | 一個可以上線的程式 |
| 加工失敗 | 煮焦了、調味錯了 | Build 失敗（程式碼有錯） |

你可能聽過「Build 失敗」（Build Failed）——就像你照著食譜做菜，結果發現少了一樣食材，這道菜做不出來。工程師就得回去檢查程式碼哪裡有問題。

**造句：**「我改完程式碼了，正在跑 **Build**，等 Build 成功就可以[部署](deploy.md)到測試環境。」

## 為什麼要知道這個詞？

- 每次工程師改完程式碼要上線前，都得先 Build 成功才行
- 「Build Failed」是工程師最常見的訊息之一——表示程式碼有問題，還不能上線
- [CI/CD](ci-cd.md) 流程中，Build 是自動化的第一步：程式碼推上去 → 自動 Build → Build 成功才會繼續後面的步驟
- 不是所有程式語言都需要 Compile（例如 [JavaScript](javascript.md) 不用），但幾乎所有專案都需要 Build 的過程來打包
