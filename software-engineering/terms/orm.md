# ORM — 跟資料庫溝通的翻譯機

> **白話說：** ORM 就像一台翻譯機——你用中文講話，翻譯機幫你翻成日文跟日本人溝通。你不用學日文，也能順利點餐。

---

## 它到底是什麼？

**ORM** 是 **Object-Relational Mapping** 的縮寫（物件關聯映射），名字很嚇人，但概念很簡單：

> **ORM 讓你用寫程式的方式操作[資料庫](database.md)，不用自己寫 SQL。**

什麼是 SQL？就是跟資料庫溝通的專用語言，長這樣：

```
不用 ORM（自己寫 SQL）：
  SELECT * FROM users WHERE age > 18 ORDER BY name;
  → 你得學 SQL 語法，還要注意資安問題

用 ORM（用程式語言操作）：
  User.where(age: > 18).order(:name)
  → ORM 自動把這段翻譯成 SQL 去跟資料庫溝通
```

就像你去日本玩：

```
不用翻譯機：自己學日文 →「すみません、メニューをください」
用翻譯機：對著手機說「不好意思，請給我菜單」→ 翻譯機幫你講日文
```

ORM 就是程式設計師的翻譯機——你用熟悉的程式語言說話，ORM 幫你翻譯成資料庫聽得懂的 SQL。

## 生活比喻 / 實際例子

| | 沒有 ORM | 有 ORM |
|---|---|---|
| 做法 | 自己手寫 SQL 語法 | 用程式語言操作 |
| 像什麼 | 自己學日文 | 帶翻譯機 |
| 優點 | 完全掌控，可以寫最精準的查詢 | 開發速度快，不用學 SQL |
| 缺點 | 要會 SQL，容易出錯 | 遇到複雜查詢可能翻譯得不夠好 |

常見的 ORM 工具：
- **Ruby on Rails** 的 ActiveRecord
- **Python** 的 SQLAlchemy、Django ORM
- **Node.js** 的 Prisma、Sequelize
- **Java** 的 Hibernate

**造句：**「我們用 Prisma 當 **ORM**，新增一筆會員資料只要寫一行程式碼，不用自己拼 SQL。」

## 為什麼要知道這個詞？

- 現代開發幾乎都會用 ORM，直接手寫 SQL 的情況越來越少
- 如果工程師說「我用 ORM 操作 DB」，就是「我用程式碼操作[資料庫](database.md)」的意思
- ORM 不是萬能的——遇到很複雜的查詢，有時候還是得自己寫 SQL 才跑得動、跑得快
- 選擇哪套 ORM 通常跟用什麼程式語言有關，是專案初期的技術決策之一
