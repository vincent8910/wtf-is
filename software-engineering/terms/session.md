# Session（工作階段） — 系統記住「你正在使用中」的那段時間

> **白話說：** 就像你走進百貨公司時拿到的一張臨時號碼牌，在你逛街的這段時間，服務台認得你是同一個人。離開後號碼牌就作廢了。

---

## 它到底是什麼？

當你登入一個網站，系統會幫你開一個「工作階段」，就是 Session。在這段期間內，網站知道你是誰，不用每按一個按鈕就重新輸入帳號密碼。

Session 有幾個重點：
- **開始**：你登入的時候
- **維持**：你在網站上操作的整段時間
- **結束**：你登出、或太久沒操作被自動踢出（session 過期）

Session 通常搭配 [Cookie](cookie.md) 或 [Token](token.md) 來運作——Cookie 是存在你瀏覽器裡的一個小記號，用來告訴伺服器「我就是剛才登入的那個人」。

## 生活比喻 / 實際例子

想像你去一家有包廂的餐廳吃飯：

- 你入座後，服務生給你一個包廂號碼（Session ID）
- 之後你點餐、加菜、叫服務，服務生都知道你是 3 號包廂的客人
- 你結帳離開後，3 號包廂就空出來給下一組客人（Session 結束）
- 如果你吃到一半去了趟廁所兩小時沒回來，包廂可能被收回去（Session 過期）

實際例子：
- 使用者說：「我用到一半就被登出了！」工程師：「**Session** 過期了，預設 30 分鐘沒操作就會自動登出」
- PM 說：「購物車裡的東西不見了」工程師：「因為 **session** 過期了，購物車資料沒有另外存」

## 為什麼要知道這個詞？

- 使用者抱怨「用到一半被登出」、「資料不見了」，很常跟 session 過期有關
- 規劃登入機制時，要決定 session 要維持多久（30 分鐘？7 天？）
- Session 跟 [Cookie](cookie.md)、[Token](token.md)、[Auth](auth.md) 都有密切關係，一起理解更完整
