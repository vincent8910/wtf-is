# Sync / Async（同步 / 非同步）— 排隊 vs 拿號碼牌

> **白話說：** 同步是排隊等結果，非同步是拿號碼牌先去做別的事，好了再來拿。

---

## 它到底是什麼？

**Sync（同步）** 和 **Async（非同步）** 是兩種處理任務的方式：

```
同步（Sync）：麥當勞排隊點餐
  → 你站在櫃檯前 → 等店員做好 → 拿到餐 → 才能離開
  → 在等的這段時間，你什麼都不能做

非同步（Async）：麥當勞拿號碼牌
  → 你點完餐拿到號碼牌 → 先去找座位、滑手機 → 叫到你的號碼再去取餐
  → 等的時候你可以做別的事
```

**同步 = 傻等；非同步 = 拿號碼牌，好了叫我。**

## 生活比喻 / 實際例子

- **同步的例子：** 你打電話給客服，電話接通前你只能聽等待音樂，什麼都不能做
- **非同步的例子：** 你用 Line 傳訊息問客服，傳完繼續做自己的事，對方回了再看

### 在軟體世界的意思

- **同步操作：** 程式送出一個請求，必須等到回應了才能繼續做下一件事
- **非同步操作：** 程式送出請求後繼續做其他事，等回應回來了再處理

### 常見情境

- **工程師：**「這個 [API](api.md) 呼叫要改成 async，不然畫面會卡住。」→ 要改成拿號碼牌模式，不要傻等
- **PM：**「匯出報表要等很久，可以做成非同步嗎？按下匯出後先做別的，好了再通知我。」
- **工程師：**「這裡是 sync 的，所以前面沒做完，後面就不會動。」

## 為什麼要知道這個詞？

當你覺得某個功能「用起來很卡」或「要等很久」，通常工程師的解法就是改成 async。理解這個概念，你就能更精準地描述問題：「這個可以做成非同步嗎？讓使用者不用傻等。」
