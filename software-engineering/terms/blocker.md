# Blocker — 擋在路中間的大石頭

> **白話說：** Blocker 就是「卡住你、讓你做不下去的事情」——不搬開就沒辦法往前走。

---

## 它到底是什麼？

想像你開車上班，前面發生車禍，路被封了。你的車沒壞，你沒遲到，但就是**過不去**。

那個車禍就是你的 **Blocker**。

在工作中也一樣：

```
前端工程師：「我要做登入畫面。」
  ↓
「但是後端的登入 API 還沒好，我沒辦法串接。」
  ↓
後端的 API 就是前端的 Blocker。
```

## 常見的 Blocker 類型

| 類型 | 例子 |
|------|------|
| **等別人的東西** | 等後端 API、等設計稿、等規格確認 |
| **等決定** | 「老闆還沒決定要不要做這個功能」 |
| **等資源** | 「測試環境還沒準備好」 |
| **技術卡關** | 「這個第三方服務的文件看不懂」 |

## 日常場景

- **「我這邊有 blocker，要等後端的 API 好才能繼續」**
  → 我被卡住了，因為需要別人先完成他的部分
- **「今天 standup 有什麼 blocker 嗎？」**
  → 站立會議上最該問的問題：誰被卡住了？
- **「幫我解一下這個 blocker」**
  → 幫我把路上的石頭搬開

> 💡 **PM 最重要的工作之一：排除 blocker。** 在 standup 裡聽到 blocker，馬上行動——去找人、去做決定、去協調資源。不然一個人被卡住，整個進度都會受影響。
