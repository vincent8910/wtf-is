# Edge Case（邊界情況）— 颱風天叫外送

> **白話說：** 一般人不太會遇到，但可能讓系統出包的極端操作或特殊狀況。

---

## 它到底是什麼？

**Edge Case** 就是那些「正常情況下不太會發生，但萬一發生了系統就可能出問題」的特殊狀況。

想像外送平台——

```
正常情況：客人點一份雞排，20 分鐘送到 ✅

Edge Case（邊界情況）：
  → 颱風天叫外送，沒有外送員接單怎麼辦？
  → 客人同時點了 100 份雞排，系統撐得住嗎？
  → 送到一半外送員手機沒電，訂單怎麼處理？
  → 地址打「火星」，系統會當機嗎？
```

**Edge Case 就是颱風天叫外送——平常不會發生，但發生了整個系統可能當場崩潰。**

## 生活比喻 / 實際例子

也像餐廳遇到的奇葩狀況——99% 的客人正常點餐，但偶爾會有人：
- 點 0 份餐點按送出
- 在備註欄打了一萬個字
- 用過期十年的折價券
- 同一秒按了兩次結帳

這些都是 edge case，如果軟體沒有考慮到，就可能出 [bug](bug-debug.md)。

### 常見情境

- **QA（品質測試人員）：**「這個功能的 edge case 我想到幾個：空白輸入、超長文字、特殊符號。」
- **工程師：**「這個 edge case 要不要處理？發生機率很低。」→ 這個極端狀況要不要花時間修？
- **PM：**「使用者反映了一個 bug，是我們沒想到的 edge case。」

## 為什麼要知道這個詞？

軟體的品質好不好，很大一部分取決於有沒有處理好 edge case。一般功能誰都會做，但能不能想到那些「萬一」，是資深工程師和 QA 的價值所在。當你聽到「要考慮 edge case」，就是在說「別只想正常情況，也要想那些奇怪的狀況」。
