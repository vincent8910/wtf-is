# Layer 1 / Layer 2（第一層/第二層） — Layer 1 是主區塊鏈，Layer 2 是蓋在上面加速用的

> **白話說：** Layer 1 是高速公路，Layer 2 是蓋在旁邊的快速道路來分散車流——主幹道太塞了，所以蓋了輔助道路讓大家走得更快、更便宜。

---

## 它到底是什麼？

在加密貨幣的世界裡，**Layer 1** 和 **Layer 2** 是用來描述[區塊鏈](blockchain.md)架構的兩個「層級」。

### Layer 1（第一層）= 主區塊鏈

Layer 1 就是「最底層的主鏈」，它是整個系統的根基。所有的交易最終都要在 Layer 1 上被確認和記錄。常見的 Layer 1 包括：

- **[以太坊](ethereum.md)**（Ethereum）
- **[比特幣](bitcoin.md)**（Bitcoin）
- **Solana**
- **Avalanche**

Layer 1 的好處是**安全性最高**——它有最多的節點在幫忙驗證交易，最不容易被攻擊。但缺點也很明顯：**速度慢、[Gas Fee](gas-fee.md) 貴**。就像台北市最主要的忠孝東路，安全可靠但一到尖峰時間就塞車塞到爆。

### Layer 2（第二層）= 加速用的輔助鏈

為了解決 Layer 1「又慢又貴」的問題，工程師們在 Layer 1「上面」蓋了 Layer 2。Layer 2 把大量交易先在自己這邊處理好，然後把結果「打包」回 Layer 1 做最終確認。

常見的 Layer 2 包括：
- **Arbitrum**、**Optimism**（基於 [Rollup](rollup.md) 技術）
- **zkSync**、**StarkNet**（基於零知識證明的 Rollup）
- **Base**（Coinbase 推出的 Layer 2）

| 比較 | Layer 1（如以太坊） | Layer 2（如 Arbitrum） |
|------|-------------------|---------------------|
| 速度 | 較慢（每秒約 15 筆交易） | 快很多（每秒數千筆） |
| Gas Fee | 貴（幾美金到幾十美金） | 便宜（幾分錢到幾毛錢） |
| 安全性 | 最高（自己驗證一切） | 高（最終靠 Layer 1 保障安全） |
| 比喻 | 高速公路本線 | 旁邊的快速道路 |

Layer 2 不是獨立的鏈，它依附在 Layer 1 上面。你可以把資產透過 [Bridge（跨鏈橋）](bridge.md) 從 Layer 1 搬到 Layer 2 上，享受更快更便宜的交易體驗。需要的時候，也可以把資產搬回 Layer 1。

## 生活比喻 / 實際例子

想像你要從台北去高雄：

- **Layer 1（國道一號）**：最老最穩的主幹道，但假日塞車塞到瘋掉。你一路塞在車陣中（Gas Fee 高、交易慢），但至少路是最安全的。
- **Layer 2（國道三號/快速道路）**：後來蓋的替代道路，走這條比較快、比較不塞。但它不是完全獨立的——最後你還是要接回主要道路到達目的地（Layer 2 的交易最終會回到 Layer 1 確認）。

或者像銀行轉帳：
- **Layer 1** = 跨行轉帳，通過央行清算系統，安全但手續費高、速度慢
- **Layer 2** = 同一間銀行內轉帳，因為不用經過央行，所以又快又免手續費。但最終銀行還是要跟央行對帳

**在幣圈你會聽到：**
- 「以太坊 **Layer 1** 的 [Gas Fee](gas-fee.md) 太貴了，我都搬到 **Layer 2** 上操作。」
- 「這個 DeFi 協議同時部署在 **L1** 和 **L2** 上，用 L2 比較划算。」
- 「要把 ETH 從 **Layer 1** 搬到 **Layer 2**，你需要用 [Bridge](bridge.md)。」

## 為什麼要知道這個詞？

Layer 1 和 Layer 2 是理解區塊鏈世界的基礎架構知識：

- **省錢**：同樣的操作，在 Layer 2 上做通常比 Layer 1 便宜 10-100 倍。如果你知道 Layer 2 的存在，就能省下大量 [Gas Fee](gas-fee.md)
- **看懂新聞**：「以太坊 Layer 2 TVL 突破新高」、「Arbitrum 空投」——這些新聞都跟 Layer 2 有關
- **理解擴容**：區塊鏈最大的挑戰之一就是「如何讓更多人同時使用」，Layer 2 正是目前最主流的解決方案
- **選擇平台**：越來越多 DApp（去中心化應用）部署在 Layer 2 上，知道 Layer 1 / Layer 2 的差別，才能選擇最適合你的平台
