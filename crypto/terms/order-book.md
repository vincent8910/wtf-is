# Order Book（訂單簿）— 所有買賣掛單的清單

> **白話說：** 訂單簿就像**拍賣場的出價板**——上面列著所有人想買的價格和想賣的價格，讓你一眼看清楚行情。

---

## 它到底是什麼？

Order Book（訂單簿）是[交易所](exchange-cex.md)上的一份**即時清單**，上面列出所有人掛出來的「想買」和「想賣」的訂單。

它通常長這樣（簡化版）：

**賣單（Ask / 賣方出價）— 從低到高排列：**

| 價格（USDT） | 數量（BTC） |
|-------------|------------|
| 65,200 | 0.5 |
| 65,100 | 1.2 |
| 65,050 | 0.8 |

**買單（Bid / 買方出價）— 從高到低排列：**

| 價格（USDT） | 數量（BTC） |
|-------------|------------|
| 65,000 | 2.0 |
| 64,950 | 1.5 |
| 64,900 | 3.0 |

買單最高價（65,000）和賣單最低價（65,050）之間的差距叫做**價差（Spread）**。價差越小，代表這個市場越活躍，交易越容易成交。

訂單簿的運作邏輯很簡單：

1. 你想買比特幣，掛一張「我願意用 65,000 USDT 買 1 顆 BTC」的單
2. 如果有人剛好掛了「我願意用 65,000 USDT 賣 1 顆 BTC」的單
3. 兩張單配對成功，交易完成！

如果沒有剛好匹配的價格，你的單就會掛在訂單簿上等待。或者你也可以選擇「市價單」——直接用目前最好的價格成交（但可能會有[滑價](slippage.md)）。

## 生活比喻 / 實際例子

想像你去菜市場賣一箱蘋果。你喊：「一顆 30 元！」旁邊一個客人說：「太貴了，25 元我就買。」另一個客人說：「27 元可以嗎？」

如果有一塊白板把所有買家和賣家的出價都寫上去，那塊白板就是「訂單簿」。大家看著白板上的價格，自己決定要不要接受哪個價格成交。

在股票市場裡也有一模一樣的東西——台灣的券商 App 上看到的「五檔報價」，就是一個簡化版的訂單簿。

**在幣圈你會聽到：**

- 「看一下 order book，買盤很厚，應該不會跌太多。」—— 買單很多代表有很多人在下面接。
- 「賣方掛了一堵大牆在 70,000，很難突破。」—— 訂單簿上某個價格有非常大量的賣單。
- 「這個幣的 order book 很薄，稍微買一點就會拉高價格。」—— 訂單簿的深度會影響價格穩定性。

## 為什麼要知道這個詞？

訂單簿是理解交易運作方式的基礎：

- **在交易所下單時**，了解訂單簿才知道為什麼你掛的價格有時候秒成交、有時候等半天
- **判斷市場狀態時**，看訂單簿的深度可以知道買賣雙方的力量對比
- **避免[滑價](slippage.md)時**，看訂單簿可以知道你的大額買單會不會把價格推太高

建議搭配了解[現貨交易](spot.md)、[交易所](exchange-cex.md)和[滑價](slippage.md)的概念。
