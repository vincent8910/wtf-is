# Hash（雜湊）— 把任何資料變成一串固定長度的亂碼指紋

> **白話說：** 雜湊就像**不管你寫多長的信，蓋上去的印章永遠一樣大**——任何資料丟進去，出來的都是一串固定長度的「指紋」，而且不可能從指紋反推回原始資料。

---

## 它到底是什麼？

**Hash（雜湊）** 是一種數學函數，做的事情很簡單：

**輸入任何東西 → 輸出一串固定長度的亂碼**

| 輸入 | Hash 輸出（舉例） |
|------|-----------------|
| "hello" | 2cf24dba5fb0a30e26e83b2ac5b9e29e... |
| "Hello" | 185f8db32271fe25f561a6fc938b2e26... |
| 整本哈利波特 | 7d1a54127b222502f5b79b5fb0803061... |
| "hello"（再算一次） | 2cf24dba5fb0a30e26e83b2ac5b9e29e...（一模一樣） |

幾個重要特性：

| 特性 | 說明 | 比喻 |
|------|------|------|
| 固定長度 | 不管輸入多大，輸出永遠一樣長 | 不管信多長，印章都一樣大 |
| 不可逆 | 無法從輸出反推輸入 | 看指紋無法還原出整個人 |
| 雪崩效應 | 改一個字，輸出完全不同 | "hello" 和 "Hello" 的 Hash 天差地遠 |
| 確定性 | 同樣的輸入永遠得到同樣的輸出 | 同一個人的指紋永遠不會變 |
| 抗碰撞 | 幾乎不可能找到兩個不同輸入有相同輸出 | 不會有兩個人指紋一模一樣 |

在[區塊鏈](blockchain.md)裡，Hash 無處不在：

- 每個區塊都有一個 Hash 值（這個區塊的「指紋」）
- 每個區塊還記錄了上一個區塊的 Hash（這樣就串成一條鏈了）
- [挖礦](mining.md)就是在拼命計算 Hash，找到符合條件的那個值
- 交易資料也會被 Hash 過，確保沒被竄改

## 生活比喻 / 實際例子

想像你把一本書放進碎紙機：

- 不管是一頁的小紙條還是一千頁的磚頭書，碎出來的紙屑量都固定（固定長度）
- 你無法把碎紙屑拼回原本的書（不可逆）
- 你改了書裡面的一個字，碎出來的紙屑完全不同（雪崩效應）
- 同一本書碎兩次，紙屑的結果一模一樣（確定性）

另一個比喻——用果汁機：

- 你把蘋果丟進果汁機 → 出來蘋果汁
- 你把鳳梨丟進果汁機 → 出來鳳梨汁
- 你無法把蘋果汁還原成蘋果（不可逆）
- 每次用同樣的蘋果打，味道都一樣（確定性）

在區塊鏈的實際應用：

| 應用場景 | Hash 怎麼用 |
|---------|------------|
| 區塊鏈接 | 每個區塊包含上一個區塊的 Hash，改了一個區塊，後面全部都要重算 |
| [挖礦](mining.md) | 礦工不斷嘗試不同的 [Nonce](nonce.md)，看哪個能產生符合條件的 Hash |
| 驗證資料 | 下載檔案後算一次 Hash，跟官方公布的比對，確認檔案沒被動過 |
| 密碼儲存 | 網站不存你的密碼原文，而是存 Hash 值 |

你會在這些地方聽到：

- 「每個區塊都有自己的 **Hash** 值」
- 「礦工在算 **雜湊**，找到符合條件的答案」
- 「把交易資料 **Hash** 過後就不可能被竄改」
- 「比特幣的 **Hash** 演算法是 SHA-256」

## 為什麼要知道這個詞？

Hash 是區塊鏈安全性的基石：

- **理解區塊鏈不可竄改的原因時**，Hash 是關鍵（改一個字，整串 Hash 都對不上）
- **理解挖礦時**，挖礦本質上就是不斷計算 Hash
- **理解[算力](hash-rate.md)時**，算力就是每秒能計算多少次 Hash
- **看技術文章時**，SHA-256、Keccak-256 這些都是 Hash 演算法的名字

簡單記：Hash = 資料的指紋。不管資料多大，指紋都一樣長，而且不可能造假。
