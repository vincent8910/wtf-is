# Nonce（隨機數）— 挖礦時要猜的那個「正確答案」

> **白話說：** Nonce 就像**密碼鎖的號碼，要一直轉才能碰對**——礦工不斷嘗試不同的數字，直到算出來的結果剛好符合條件。

---

## 它到底是什麼？

**Nonce** 是 **Number used once（只用一次的數字）** 的縮寫。

在[挖礦](mining.md)的過程中，礦工要做的事情是：

1. 把一堆交易資料 + 上一個區塊的 [Hash](hash.md) + 一個「猜的數字」放在一起
2. 對這堆東西做 Hash 計算
3. 看計算結果有沒有符合條件（比如開頭要有 20 個零）
4. 不符合？換一個數字再算一次
5. 一直換、一直算，直到找到那個讓結果剛好符合的數字

**這個「猜的數字」就是 Nonce。**

| 過程 | 說明 |
|------|------|
| 第 1 次嘗試 | 交易資料 + Nonce = 0 → Hash 結果：8a3f2b...（不對） |
| 第 2 次嘗試 | 交易資料 + Nonce = 1 → Hash 結果：c7d91e...（不對） |
| 第 3 次嘗試 | 交易資料 + Nonce = 2 → Hash 結果：f12a8b...（不對） |
| ... | 可能要試幾十億次... |
| 第 N 次嘗試 | 交易資料 + Nonce = 38472910 → Hash 結果：000000000a1b2c...（符合！） |

找到正確的 Nonce = 挖到了！你可以把這個區塊寫進[區塊鏈](blockchain.md)，拿到[區塊獎勵](block-reward.md)。

為什麼找 Nonce 這麼難？因為 [Hash](hash.md) 的特性——你改一個數字，結果就完全不同，沒有規律可循。就像開密碼鎖一樣，只能一個一個試。

## 生活比喻 / 實際例子

**比喻一：密碼鎖**

想像一把四位數的密碼鎖（0000 ~ 9999）：

- 你不知道密碼是什麼
- 只能從 0000 開始，一個一個試
- 0000？不對。0001？不對。0002？不對...
- 試到 3847？叩！開了！

3847 就是這把鎖的「Nonce」。只不過在[比特幣](bitcoin.md)的世界裡，密碼鎖不是四位數，而是幾十位數，所以需要超強的[算力](hash-rate.md)來猜。

**比喻二：摸彩球**

想像一個箱子裡有 10 億顆球，只有一顆是金色的：

- 你每秒從箱子裡摸一顆球出來看
- 不是金色的？放回去，再摸一顆
- 摸到金色的那一刻 = 找到 Nonce 了！
- [算力](hash-rate.md)越高 = 每秒能摸越多顆球

在挖礦的實際運作中：

- [比特幣](bitcoin.md)的 Nonce 是一個 32 位元的數字（最多約 43 億種可能）
- 但光靠 Nonce 可能不夠，礦工還會調整其他欄位來產生更多組合
- 全球礦工每秒嘗試幾百萬兆次（幾百 EH/s）
- 平均每 10 分鐘會有一個礦工幸運地找到正確答案

你會在這些地方聽到：

- 「礦工不斷嘗試不同的 **Nonce** 來挖礦」
- 「找到正確的 **Nonce** 就能打包區塊」
- 「**Nonce** 的搜尋過程就是工作量證明」
- 「這個區塊的 **Nonce** 值是 2,859,274,016」

## 為什麼要知道這個詞？

Nonce 是理解 [PoW 挖礦](proof-of-work.md)原理的核心拼圖：

- **理解挖礦在做什麼時**，挖礦就是在猜 Nonce
- **理解為什麼挖礦要用那麼多電時**，因為要猜的次數太多了
- **理解難度調整時**，難度越高，符合條件的 Nonce 越難找
- **看技術文章時**，Nonce 是區塊頭（Block Header）裡的重要欄位之一

簡單記：Nonce = 挖礦時要猜的那個幸運數字。猜對了就中獎。
