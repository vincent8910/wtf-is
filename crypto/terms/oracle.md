# Oracle（預言機） — 把現實世界的資料餵給區塊鏈的服務

> **白話說：** 區塊鏈的「新聞記者」——區塊鏈自己看不到外面的世界，所以需要有人專門負責把外面發生的事（股價、天氣、比賽結果）告訴鏈上的程式。

---

## 它到底是什麼？

**Oracle（預言機）** 是一種把「現實世界的資料」傳送到[區塊鏈](blockchain.md)上的服務。

為什麼需要 Oracle？因為區塊鏈有一個天生的限制：**它看不到自己以外的世界**。區塊鏈只知道鏈上發生的事情（誰轉帳給誰、誰 [Mint](mint.md) 了什麼 [NFT](nft.md)），但它不知道：

- 現在台積電的股價是多少？
- 今天台北的氣溫是幾度？
- 昨天世界盃誰贏了？
- 現在 1 美金可以換多少台幣？

這些「鏈外資料」，區塊鏈自己查不到。這時候就需要 Oracle 來當「中間人」，把外面世界的資料搬到鏈上。

```
現實世界                    Oracle（預言機）              區塊鏈

台積電股價：$180  ──────→                 ──────→   智能合約收到：
黃金價格：$2,300  ──────→   收集、驗證、    ──────→   台積電 = $180
台北氣溫：22°C   ──────→   傳送資料       ──────→   黃金 = $2,300
                                                    台北 = 22°C
```

最有名的 Oracle 項目是 **Chainlink**，它不是只有一個人在報數據，而是有一整個網路的「節點」在蒐集和驗證資料。這樣做是為了防止有人餵假資料——如果只有一個人報數據，那個人說謊怎麼辦？所以 Oracle 通常會讓很多人同時報，取中間值或多數決。

Oracle 最常被用在 **DeFi（去中心化金融）** 領域：

| 應用場景 | 需要什麼資料 | Oracle 提供 |
|---------|------------|-----------|
| 借貸平台 | 抵押品現在值多少？ | 即時幣價 |
| 去中心化交易所 | 各種代幣的匯率？ | 即時匯率 |
| 保險合約 | 航班有沒有延誤？ | 航班狀態 |
| 預測市場 | 選舉結果是什麼？ | 選舉結果 |

## 生活比喻 / 實際例子

把 Oracle 想成你公司的「外勤人員」：

- **辦公室裡的人（區塊鏈上的智能合約）**：坐在辦公室裡，只看得到辦公室裡面的資料
- **外勤人員（Oracle）**：在外面跑來跑去，收集各種資訊回報給辦公室
- **辦公室的人根據外勤回報做決策**：「外勤說客戶已經付款了，那我們就出貨」

或者更生活化的比喻：

想像你在家裡用手機 App 賭運動比賽結果。App 上的「賭局智能合約」不知道比賽結果——它只是一段程式碼。所以需要有人（Oracle）在比賽結束後告訴它：「結果出來了，是 A 隊贏」，合約才能自動把獎金發給賭對的人。

如果 Oracle 回報假的結果呢？那所有人的錢就會被錯誤地分配。這就是為什麼 Oracle 的「可信度」這麼重要——它就像法庭上的證人，如果證人說謊，判決就會出錯。

**在幣圈你會聽到：**
- 「這個 DeFi 協議用 Chainlink 的 **Oracle** 取得即時幣價。」
- 「**預言機** 被攻擊了，駭客餵了假的價格資料，導致借貸平台被掏空。」
- 「沒有可靠的 **Oracle**，DeFi 根本無法運作。」

## 為什麼要知道這個詞？

Oracle 是區塊鏈世界裡一個容易被忽略、但極其重要的基礎設施：

- **理解 DeFi 的運作**：幾乎所有 DeFi 應用都依賴 Oracle 來取得外部數據。沒有 Oracle，借貸、交易、保險等功能都無法運作
- **理解安全風險**：「Oracle 攻擊」是 DeFi 被駭最常見的原因之一。駭客操縱 Oracle 餵給合約的價格，就能從中獲利。了解 Oracle，才能理解為什麼有些 DeFi 協議會被攻擊
- **看懂區塊鏈的侷限**：區塊鏈不是萬能的，它需要 Oracle 才能跟現實世界互動。知道這點，你對區塊鏈的理解就會更全面
