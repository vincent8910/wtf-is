# Block Reward（區塊獎勵）— 成功記帳後獲得的加密貨幣獎勵

> **白話說：** 區塊獎勵就像**搶答成功的獎金**——你的電腦搶先完成記帳工作，系統就自動發加密貨幣給你當作報酬。

---

## 它到底是什麼？

**Block Reward（區塊獎勵）** 是[區塊鏈](blockchain.md)系統發給成功記帳者的[加密貨幣](cryptocurrency.md)獎勵。

在 [PoW（工作量證明）](proof-of-work.md)的區塊鏈裡，流程是這樣的：

1. 礦工們拼命計算，爭取記帳權（[挖礦](mining.md)）
2. 某個礦工先找到正確的 [Nonce](nonce.md)
3. 他把這批交易打包成一個區塊，寫進區塊鏈
4. 系統自動「憑空生成」一筆加密貨幣，發到他的錢包裡
5. 這筆「憑空生成」的幣 = 區塊獎勵

區塊獎勵其實包含兩個部分：

| 組成 | 說明 |
|------|------|
| 新幣獎勵 | 系統新產生的加密貨幣（這是新幣進入流通的方式） |
| 交易手續費 | 這個區塊裡所有交易附帶的手續費 |

以[比特幣](bitcoin.md)為例，區塊獎勵的歷史變化：

| 時期 | 區塊獎勵（新幣部分） | 備註 |
|------|-------------------|------|
| 2009–2012 | 50 BTC | 最早期 |
| 2012–2016 | 25 BTC | 第一次[減半](halving.md) |
| 2016–2020 | 12.5 BTC | 第二次減半 |
| 2020–2024 | 6.25 BTC | 第三次減半 |
| 2024–2028 | 3.125 BTC | 第四次減半 |

隨著[減半](halving.md)機制，新幣獎勵越來越少。最終在 2140 年左右，新幣獎勵會降到零，礦工的收入就只剩交易手續費了。

## 生活比喻 / 實際例子

想像學校辦了一個搶答比賽：

1. 老師出一道題，全班搶答
2. **搶答成功的同學**獲得：
   - **獎金 100 元**（= 新幣獎勵）
   - **幫同學寫作業的酬勞 20 元**（= 交易手續費）
3. 所以這次搶答的「區塊獎勵」= 100 + 20 = **120 元**
4. 但是！**每學期獎金砍一半**：
   - 第一學期：100 元
   - 第二學期：50 元
   - 第三學期：25 元...

在現實世界裡：

- 2009 年挖到一個比特幣區塊能拿 50 BTC，當時幾乎不值錢
- 同樣 50 BTC 放到現在價值超過幾百萬美元
- 現在的區塊獎勵是 3.125 BTC，雖然數量少了，但價值可能比以前高

你會在這些地方聽到：

- 「比特幣的 **區塊獎勵** 現在是 3.125 BTC」
- 「減半後 **Block Reward** 會砍一半，礦工壓力很大」
- 「**區塊獎勵** 是新比特幣進入流通的唯一方式」
- 「當 **區塊獎勵** 歸零後，礦工只靠手續費維生」

## 為什麼要知道這個詞？

區塊獎勵是理解加密貨幣經濟模型的關鍵：

- **理解比特幣怎麼「產生」時**，區塊獎勵就是新幣的來源
- **理解[減半](halving.md)的影響時**，減半直接砍掉一半的區塊獎勵
- **評估挖礦收益時**，區塊獎勵是礦工的主要收入
- **思考比特幣的未來時**，當區塊獎勵歸零，網路靠什麼維持運作？

簡單記：區塊獎勵 = 記帳成功的獎金。這也是新的加密貨幣誕生的方式。
