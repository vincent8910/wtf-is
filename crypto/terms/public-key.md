# Public Key（公鑰）— 可以公開給別人的收款地址來源

> **白話說：** 公鑰就像你的**銀行帳號**——別人知道也沒關係，知道了才能轉錢給你。

---

## 它到底是什麼？

Public Key（公鑰）是由[私鑰](private-key.md)透過數學運算產生出來的一串字串。你可以把它想成一個「從私鑰衍生出來的公開資訊」。

重點來了：**從私鑰可以算出公鑰，但從公鑰絕對無法反推出私鑰。** 這就像你可以從雞蛋做出蛋糕，但沒辦法從蛋糕還原出雞蛋。這個單向的數學特性，就是加密貨幣安全性的基礎。

整個關係鏈是這樣的：

```
私鑰 → 公鑰 → 地址
（印章）  （身分證字號）  （銀行帳號）
```

| 步驟 | 說明 | 類比 |
|------|------|------|
| 1. [私鑰](private-key.md) | 系統隨機產生 | 你的印章 |
| 2. 公鑰 | 由私鑰計算出來 | 你的身分證字號 |
| 3. [地址](address.md) | 由公鑰再計算出來 | 你的銀行帳號 |

在日常使用中，你其實很少會直接看到或用到「公鑰」這個東西——你比較常接觸到的是「[地址](address.md)」（由公鑰產生的收款字串）。但了解公鑰的存在，有助於你理解整個加密貨幣系統是怎麼保障安全的。

公鑰的另一個重要功能是**驗證簽名**。當你用私鑰「簽名」一筆交易後，其他人可以用你的公鑰來確認：「這筆交易確實是這個人授權的。」就像你蓋了印章，別人可以拿你的印鑑證明來核對。

## 生活比喻 / 實際例子

你去銀行開戶，銀行給你一個帳號（公鑰/地址）。你可以把帳號給任何人：「喂，轉帳到這個帳號就好。」對方知道你的帳號，能匯錢給你，但他沒辦法光靠帳號就把你的錢領走——他還需要你的存摺和印章（私鑰）才行。

公鑰就是這個概念：**可以安全地公開，別人拿到也只能匯錢給你，不能偷走你的錢。**

**在幣圈你會聽到：**

- 「把你的收款地址傳給我就好。」—— 地址是由公鑰衍生的，可以放心給別人。
- 「公鑰跟私鑰是一對的，公鑰可以公開，私鑰絕對不能。」—— 這是加密貨幣安全的基本觀念。
- 「區塊鏈用公鑰來驗證這筆交易是不是你本人發的。」—— 這是公鑰在技術上的核心用途。

## 為什麼要知道這個詞？

公鑰雖然不像[私鑰](private-key.md)或[助記詞](seed-phrase.md)那麼需要你「保護」，但理解它有助於你搞懂整個系統：

- **理解安全性時**，知道「公鑰不能反推私鑰」這個特性，你就懂為什麼加密貨幣是安全的
- **跟人解釋時**，公鑰 vs 私鑰的概念是你進入幣圈的基礎知識
- **遇到問題時**，了解私鑰 → 公鑰 → 地址的關係，可以幫助你排除很多困惑

了解公鑰之後，建議搭配閱讀[私鑰](private-key.md)和[地址](address.md)，把整套密碼學關係搞清楚。
