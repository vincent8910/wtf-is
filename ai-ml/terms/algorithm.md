# Algorithm（演算法） — 解決問題的一套步驟和規則

> **白話說：** 就像泡麵的烹煮說明——「1. 煮水 2. 放麵 3. 等三分鐘 4. 加調味包」。演算法就是告訴電腦「按照什麼步驟解決問題」。

---

## 它到底是什麼？

Algorithm（演算法）就是一套有明確步驟的解題流程。

電腦不會自己想辦法，你必須給它一個清楚的指令流程。這個流程就叫演算法。不只在 AI 裡，所有電腦程式的核心都是演算法。

在 AI 的領域，演算法指的是 [Machine Learning](machine-learning.md) 用來「從資料中學習」的方法。不同的演算法適合不同的問題：

- 想把東西分類（這是貓 or 狗）→ 用 [Classification](classification.md) 演算法
- 想預測數字（明天股價多少）→ 用 [Regression](regression.md) 演算法
- 想找出資料裡的隱藏分群 → 用 [Unsupervised Learning](unsupervised-learning.md) 演算法

## 生活比喻 / 實際例子

你每天早上出門上班，其實也在執行一個「演算法」：

1. 看天氣預報 → 決定穿什麼
2. 看時間 → 決定搭捷運還是騎車
3. 如果遲到了 → 叫 Uber
4. 到公司 → 完成

這就是你的「通勤演算法」。不同人有不同的演算法，有的人效率高（5 分鐘出門），有的人效率低（1 小時才出門）。

實際會聽到的說法：
- 「YouTube 的推薦 **演算法** 又改了，流量掉好多」
- 「這個 **Algorithm** 的準確率有 95%」
- 「我們需要換一個更好的 **演算法** 來處理這個問題」

## 為什麼要知道這個詞？

「演算法」是最基礎的 AI / 科技詞彙之一。每次有人說「平台的演算法」（像 IG、YouTube、TikTok），他們其實在說「平台背後用什麼規則來決定你看到什麼內容」。了解這個詞，你就不會被唬住。
