# User Journey（使用者旅程） — 把使用者從頭到尾的體驗過程畫成一張地圖

> **白話說：** 就像畫一張「客人從走進百貨公司到離開」的完整動線圖，記錄每一步他做了什麼、心情怎樣、哪裡開心、哪裡想罵人。

---

## 它到底是什麼？

User Journey（也叫 User Journey Map）是把使用者「從認識到使用你的產品」的整個過程，一步一步畫出來。

這張地圖上會標示：
- **每個步驟**：使用者做了什麼（搜尋、註冊、下單⋯⋯）
- **每步的感受**：開心、困惑、焦慮、生氣
- **接觸點**：在哪裡跟你的產品互動（網站、App、客服⋯⋯）
- **[痛點](pain-point.md)**：哪裡讓人覺得不爽

畫 User Journey 的目的是：**用使用者的角度看問題，而不是用公司的角度。**

## 生活比喻 / 實際例子

想像你用 Uber Eats 訂午餐的完整旅程：

| 步驟 | 你做了什麼 | 心情 |
|------|-----------|------|
| 1. 打開 App | 滑手機、打開 Uber Eats | 還好 |
| 2. 找餐廳 | 滑了一堆看起來都差不多的店 | 有點煩 |
| 3. 選餐點 | 猶豫半天，加購飲料 | 還行 |
| 4. 結帳 | 發現運費 60 元 | 有點不爽 |
| 5. 等待 | App 顯示 40 分鐘，一直盯著看 | 焦慮 |
| 6. 拿到餐 | 外送員送到、餐是熱的 | 開心 |
| 7. 吃完 | 覺得還不錯 | 滿足 |

這就是一個簡單的 User Journey。Uber Eats 的團隊看到「第 4 步心情下降」，就可能推出免運費活動來改善。

實際場景：
- PM：「我們畫一下 **User Journey**，看看使用者在哪一步流失最多。」
- 設計師：「從 **User Journey** 來看，註冊流程太長了，很多人中途放棄。」
- 行銷：「**User Journey** 顯示使用者在比價階段會去看競品，我們得加強這個環節。」

## 為什麼要知道這個詞？

- 畫出 User Journey 才能看到「全貌」——不只是某個按鈕好不好按，而是整個體驗順不順
- 幫助團隊從使用者角度思考，而不是從「我們有什麼功能」出發
- 常跟 [Pain Point（痛點）](pain-point.md)、[User Research（使用者研究）](user-research.md) 一起出現
- 不管你是 PM、設計師、行銷，看到 User Journey Map 就知道「原來使用者是這樣經歷的」
