# User Research（使用者研究） — 先搞懂你的使用者到底要什麼，再開始做產品

> **白話說：** 就像開餐廳之前，先去問附近居民平常喜歡吃什麼、什麼時間會出來吃飯，而不是自己覺得好吃就開賣。

---

## 它到底是什麼？

User Research 就是「去了解你的使用者」的過程。

不管是做 App、做網站、還是設計一個服務，在動手之前，先透過訪談、問卷、觀察等方式，搞清楚使用者的需求、習慣、困擾。

這件事的重點是：**你以為的 ≠ 使用者真正想要的**。

很多產品失敗不是因為技術不好，而是因為做了一個「沒人需要的東西」。User Research 就是為了避免這件事。

常見的研究方法：
- **訪談**：一對一聊，問使用者平常怎麼做
- **問卷**：大量收集意見
- **觀察**：看使用者實際怎麼操作
- **[Usability Testing](usability-testing.md)**：請使用者試用產品，看哪裡卡住

## 生活比喻 / 實際例子

想像你要在學校旁邊開一家飲料店：

- **沒做 User Research**：「我覺得大家都愛喝珍奶，所以只賣珍奶！」→ 結果附近學生都在減糖，都跑去買隔壁的無糖綠
- **有做 User Research**：先去學校門口問 50 個學生平常喝什麼、預算多少、什麼時間買 → 發現大家偏好低糖飲料、價位 40-60 元、放學後最常買

實際場景：
- PM 說：「先做 **User Research** 再寫規格，不要自己腦補。」
- 設計師：「我們上週做了 **User Research**，發現使用者根本找不到結帳按鈕。」
- 老闆問：「這個功能是根據什麼決定的？」團隊：「根據 **User Research** 的訪談結果。」

## 為什麼要知道這個詞？

- 這是所有 UX 設計的起點——不做研究就做設計，等於閉著眼睛開車
- 開會時聽到「我們先做 User Research」，代表團隊要先去了解使用者，不是馬上開始畫設計
- 跟 [Pain Point（痛點）](pain-point.md)、[User Journey（使用者旅程）](user-journey.md) 經常一起出現
- 不管你是工程師、行銷還是老闆，了解這個詞就知道「為什麼不能馬上動工」
