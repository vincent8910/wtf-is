# Responsive Design（響應式設計） — 同一個網頁，手機、平板、電腦看起來都合適

> **白話說：** 就像一件彈性布料的衣服——不管高矮胖瘦穿上去都合身。網頁也是，不管螢幕大小，內容都會自動調整排列，看起來都很舒服。

---

## 它到底是什麼？

Responsive Design 是指一個網頁能「自動適應不同螢幕尺寸」的設計方式。

同一個網頁：
- **手機上**：內容變成一欄、按鈕變大、選單收進漢堡圖示（三條線）
- **平板上**：內容變成兩欄、圖片和文字並排
- **電腦上**：完整三欄、導覽列全部展開

使用者不需要特別放大縮小、左右滑動，內容就自動排好了。

為什麼需要 Responsive Design？因為現在超過一半的人都用手機上網。如果你的網頁在手機上看起來文字超小、按鈕擠成一團，使用者直接就走了。

## 生活比喻 / 實際例子

想像一張自動調整的餐桌：

- **只有你一個人吃**：桌子縮成小茶几，剛好放一份餐
- **兩個人吃**：桌子展開一些，面對面坐
- **家族聚餐 10 個人**：桌子全部展開，大家圍在一起

內容（菜）都是同一套，但桌子（版面）會自動調整到最適合的大小。

實際例子：
- 打開 YouTube 的網站——在電腦上是三排影片並列、在手機上自動變成一排往下滑
- 蝦皮的商品頁，電腦上圖片跟文字是左右排列，手機上變成上下排列

實際場景：
- 設計師：「這個網頁要做 **Responsive Design**，手機版的按鈕要放大。」
- PM：「我們的用戶 70% 用手機，**RWD** 一定要做好。」
- 客戶反映：「你們的網頁用手機看文字好小！」→ 八成是 **Responsive Design** 沒做好

## 為什麼要知道這個詞？

- 現在做網站，Responsive Design 已經是基本要求，不是加分項目
- 聽到 RWD（Responsive Web Design 的縮寫）就是同一回事
- Google 也會把「手機體驗好不好」列入搜尋排名的考量——不做 RWD 連搜尋排名都會掉
- 跟 [Information Architecture（資訊架構）](information-architecture.md) 有關：不同螢幕上要決定哪些內容先顯示、哪些先隱藏
