## 這是什麼？

這是一份給**沒有 UX 或產品設計背景的人**看的術語翻譯手冊。

如果你是工程師、行銷、老闆，或任何在產品會議上聽到 User Journey、Pain Point、PRD 就一頭霧水的人——這裡就是你的救星。

每個術語都有：
- **白話解釋**：不用任何 UX 背景就能懂
- **生活比喻**：用你熟悉的事物來對照
- **造句示範**：讓你知道這個詞在什麼情境下會出現

## 使用者研究類

> 做產品之前先搞懂使用者——不做研究就做設計，等於閉著眼睛開車。

| 詞彙 | 一句話解釋 | 比喻 | 你會在哪聽到 | 詳細說明 |
|------|-----------|------|-------------|---------|
| User Research（使用者研究） | 先了解使用者要什麼，再開始做產品 | 開餐廳前先問附近居民喜歡吃什麼 | 產品規劃、設計初期 | [詳細](terms/user-research.md) |
| Usability Testing（可用性測試） | 請真人來試用，看哪裡卡住 | 餐廳試營運，看客人點餐會不會卡住 | 設計驗證、上線前測試 | [詳細](terms/usability-testing.md) |
| User Journey（使用者旅程） | 把使用者從頭到尾的體驗畫成地圖 | 畫一張「客人從進百貨到離開」的動線圖 | 產品設計、體驗優化 | [詳細](terms/user-journey.md) |
| Pain Point（痛點） | 使用者覺得最不爽的地方 | 看醫生問你哪裡不舒服，那就是痛點 | 幾乎所有產品會議 | [詳細](terms/pain-point.md) |
| Heuristic Evaluation（啟發式評估） | 專家用檢查清單幫設計做健康檢查 | 請老師傅來看裝潢，靠經驗一眼抓出問題 | 設計審查、品質檢測 | [詳細](terms/heuristic-evaluation.md) |

## 設計方法類

> 好的設計不是靠靈感，而是有方法、有原則的。

| 詞彙 | 一句話解釋 | 比喻 | 你會在哪聽到 | 詳細說明 |
|------|-----------|------|-------------|---------|
| Design Thinking（設計思考） | 先搞懂問題，再想解決方案 | 醫生先問診再開藥，不是一進門就開刀 | 產品規劃、創新工作坊 | [詳細](terms/design-thinking.md) |
| Information Architecture（資訊架構） | 決定東西怎麼分類、放在哪裡 | 超市貨架怎麼擺，客人才能快速找到醬油 | 網站改版、App 架構規劃 | [詳細](terms/information-architecture.md) |
| Interaction Design（互動設計） | 設計使用者跟產品怎麼互動 | 電梯按鈕怎麼設計，不用看說明書就會搭 | UI 設計、產品開發 | [詳細](terms/interaction-design.md) |
| Affordance（操作暗示） | 讓人一看就知道能按、能滑、能拉 | 門把形狀告訴你要推還是拉 | 設計審查、UI 設計 | [詳細](terms/affordance.md) |
| Responsive Design（響應式設計） | 同一個網頁在手機、平板、電腦都合適 | 彈性布料的衣服，高矮胖瘦穿起來都合身 | 網頁開發、設計規格 | [詳細](terms/responsive-design.md) |

## 產品策略類

> 做什麼、不做什麼、先做哪個——這就是產品策略在決定的事。

| 詞彙 | 一句話解釋 | 比喻 | 你會在哪聽到 | 詳細說明 |
|------|-----------|------|-------------|---------|
| Product Roadmap（產品路線圖） | 產品接下來要做什麼的時間規劃表 | 旅行計畫表：第一天去九份、第二天去花蓮 | 產品會議、跨部門同步 | [詳細](terms/product-roadmap.md) |
| Feature Prioritization（功能優先排序） | 決定先做哪個功能、後做哪個 | 預算有限，先買冬天外套還是新球鞋？ | Sprint 規劃、產品排程 | [詳細](terms/feature-prioritization.md) |
| User Story（使用者故事） | 用一句話描述誰想做什麼、為什麼 | 點餐跟店員說你是誰、想吃什麼、為什麼 | Agile 會議、需求討論 | [詳細](terms/user-story.md) |
| PRD（產品需求文件） | 把「要做什麼」寫清楚的說明書 | 裝潢前寫的詳細需求單 | 開發啟動前、需求審查 | [詳細](terms/prd.md) |
| North Star Metric（北極星指標） | 全公司最重要的那一個數字 | 航海靠北極星確認方向，不管風浪怎麼變 | 產品策略、全公司目標 | [詳細](terms/north-star-metric.md) |

## 比喻對照總表

> 一次看完所有比喻，讓你對 UX / 產品設計術語建立直覺。

### 使用者研究類

| 術語 | 比喻 |
|------|------|
| User Research | 開餐廳前先問附近居民喜歡吃什麼，而不是自己覺得好吃就開賣 |
| Usability Testing | 餐廳試營運，請朋友來吃看看會不會卡住 |
| User Journey | 畫一張「客人從進百貨到離開」的完整動線圖，記錄每步心情 |
| Pain Point | 看醫生問你「哪裡不舒服？」——那個不舒服的地方就是痛點 |
| Heuristic Evaluation | 請有經驗的老師傅來看裝潢，靠經驗一眼就看出問題 |

### 設計方法類

| 術語 | 比喻 |
|------|------|
| Design Thinking | 醫生看病：先問診、做檢查、再開藥方、最後追蹤效果 |
| Information Architecture | 超市貨架怎麼擺——泡麵、飲料、生鮮各自在哪個區域 |
| Interaction Design | 電梯按鈕怎麼設計，不用看說明書就知道怎麼搭 |
| Affordance | 門把手的形狀告訴你要推還是拉，不用貼說明書 |
| Responsive Design | 彈性布料的衣服，不管高矮胖瘦穿上去都合身 |

### 產品策略類

| 術語 | 比喻 |
|------|------|
| Product Roadmap | 旅行計畫表——第一天九份、第二天花蓮、第三天墾丁 |
| Feature Prioritization | 預算有限，先買冬天外套還是新球鞋？排出先後順序 |
| User Story | 點餐時跟店員說：我是上班族、想快速吃飽、因為午休只有一小時 |
| PRD | 裝潢前的詳細需求單：客廳木地板、牆壁淺灰、插座裝這個位置 |
| North Star Metric | 航海靠北極星辨方向——一個數字讓全公司對準同一個方向 |
